## 摘要

2024 年是开源 LLM 的"双峰年":Meta 用纯蛮力推出 LLaMA-3 系列(8B/70B/405B),DeepSeek 用极致工程推出 V2/V3(236B-MoE / 671B-MoE)。两者代表两条截然不同的路线:

- **LLaMA-3**:dense + GQA + RoPE + bf16 + 15T 数据,**简单可靠的"配方升级"**
- **DeepSeek-V3**:MoE + MLA + MTP + FP8 + DualPipe,**架构与系统层全栈创新**

本章把两者按"架构 / 训练 / 推理 / Benchmark / 哲学 / 影响力"五维拉直对照,帮你建立选型直觉。两者并非简单的"谁强谁弱",而是**针对不同问题最优**——读完你能解释为什么这两条路线都正确,以及何时应该选哪条。

## 27.1 对比口径

(LLaMA-3.3 70B 等也会顺带提到。R1 路线放 §28 单独讲。)

数字主要来源:Meta 论文 *"The Llama 3 Herd of Models"* + DeepSeek-V3 Technical Report。

## 27.2 架构对比一览

| 模块 | LLaMA-3-405B | DeepSeek-V3 671B |
| --- | --- | --- |
| 范式 | **Dense** | **MoE**(256 routed + 1 shared,Top-8) |
| 层数 | 126 | 61 |
| hidden | 16384 | 7168 |
| FFN 隐藏维 | 53248 | 18432(每专家 2048) |
| Heads | 128 | 128 |
| Attention | **GQA**(8 group) | **MLA**(KV 压 ~57×) |
| 位置编码 | RoPE(YaRN 类扩展) | RoPE + decoupled + YaRN |
| 激活 | SwiGLU | SwiGLU |
| Norm | RMSNorm,Pre-Norm | RMSNorm,Pre-Norm |
| 训练精度 | **bf16** | **bf16 + FP8(矩阵乘)** |
| MTP head | ❌ | ✅(1 个,辅助 + speculative) |
| Tie weights | ❌ | ❌ |

### 27.2.1 关键差异

- **Dense vs MoE**:LLaMA-3 走"通用 scaling",所有 token 全过所有参数;V3 用 MoE,每 token 只过 5-6% 参数
- **GQA vs MLA**:LLaMA-3 KV 8 组共享(压 8×);V3 用 MLA(压 ~57×),长上下文显存差距是数量级
- **bf16 vs FP8**:LLaMA-3 保守,V3 激进——V3 训练吞吐 ~1.6× 与显存 ~0.6× 全在这里

## 27.3 训练对比一览

| 项 | LLaMA-3-405B | DeepSeek-V3 |
| --- | --- | --- |
| 训练 token | 15.6 T | 14.8 T |
| 上下文(主训) | 8k | 4k |
| 上下文扩展 | 128k(主训阶段就长) | 128k(YaRN 扩) |
| 训练硬件 | ~16000 H100 | ~2048 H800 |
| 训练时长 | ~ 54 天 | ~ 2 个月 |
| GPU 小时 | ~ 30.8 M H100-hr | ~ 2.79 M H800-hr |
| 训练 FLOPs(粗算 6PD) | ~ $3.8\times 10^{25}$ | ~ $3.3\times 10^{24}$(按激活算) |
| 训练成本估算 | ~ \$60 M+ | ~ \$5.5 M |
| 并行 | TP/PP/DP/SP | PP/EP/DP(无 TP) |
| 调度 | 1F1B + 标准 | **DualPipe** |
| 优化器 | AdamW | AdamW |
| 数据混合 | 类 LLaMA-2 + 代码强化 | 中英 + 代码 + 数学,3 阶段课程 |

### 27.3.1 训练成本差近 10×

来源拆解:

| 因素 | 贡献 |
| --- | --- |
| MoE(每 token 算力 ~10×↓) | 4-6× |
| FP8 矩阵乘(吞吐 ~1.6×↑) | 1.6× |
| DualPipe + 通信重叠 | 1.2× |
| H800 单价低于 H100 | 1.2× |
| 训练 token 略少 | 1.05× |

乘起来 ~ 10× 的成本差,与公开数字大体一致。

### 27.3.2 关键观察

- LLaMA-3 走"超 Chinchilla 过训练"(D/P ≈ 38),数据驱动
- V3 走"MoE + 系统优化",架构与系统驱动
- 两条路线**等效收益**但成本结构完全不同:LLaMA-3 把钱花在"更多算力 + 更多数据";V3 把钱花在"更聪明的架构 + 更激进的系统"

## 27.4 推理与部署对比

| 项 | LLaMA-3-405B | DeepSeek-V3 |
| --- | --- | --- |
| 单 token 激活算力 | 全 405B | **37B**(MoE 稀疏) |
| 单 token KV(bf16) | ~ 64 KB(GQA) | **~ 1.1 KB**(MLA) |
| 128k 上下文 KV | ~ 8 GB | **~ 0.14 GB** |
| 单卡可部署 | ❌(必须多卡) | 多卡(EP 主导) |
| 8× H100 部署 | 极致量化勉强 | **轻松 + 多并发** |
| 推理 FLOPs / token | 较高 | **较低**(只激活 37B) |
| 商业 API 定价(典型) | 中等-高 | **极低** |

### 27.4.1 单 query 推理速度

短 prompt + 短输出场景:

- LLaMA-3-405B:每 token 跑 ~ 405B 参数 → 慢
- V3:每 token ~ 37B 激活 → 快得多

虽然 V3 总模型大,但 **decode 是 memory-bound,主要由激活参数决定带宽访问**。V3 用 1/10 的激活参数 → 单 query latency 显著低于 405B。

### 27.4.2 高并发吞吐

- LLaMA-3:KV cache 中等大(GQA);并发 batch 受 KV 限制
- V3:KV cache 极小(MLA),**同一卡同时服务的并发数显著高**

这就是为什么 DeepSeek 自家 API 在 2025 年初能开出业界最低的 token 单价。

### 27.4.3 端侧 / 单卡

- LLaMA-3-405B:几乎不可能在单卡(包括 H100)上跑
- V3:总参 671B,**单卡部署也不可能**;但 V3 系列里有 V3-Small / 蒸馏小模型可上单卡

对端侧 / 边缘场景,**两者都不直接适合**——这是 LLaMA-3-8B / Qwen-7B / DeepSeek-R1 蒸馏小模型的舞台。

(取公开报告 + 第三方评测的大致水位,具体数字按发布更新)

| Benchmark | LLaMA-3.1-405B | DeepSeek-V3 | GPT-4o(参考) |
| --- | --- | --- | --- |
| MMLU(5-shot) | 87.3 | **88.5** | ~88 |
| MMLU-Pro | 73.3 | **75.9** | 74 |
| MATH | 73.8 | **75.7** | 76.6 |
| GPQA Diamond | 51.1 | **59.1** | 53.6 |
| HumanEval | 89.0 | **89.0** | 90.2 |
| MBPP | 78.8 | **79.4** | 80 |
| SWE-Bench Verified | ~16 | ~28 | ~25 |
| GSM8K(Chain-of-Thought) | 96.8 | **96.5** | 95 |
| 中文(C-Eval / CMMLU) | 一般 | **强** | 中等 |
| 长上下文(LongBench) | 接近 SOTA | **SOTA** | SOTA |

### 27.5.1 总体结论

- **学术 benchmark 上 V3 略胜或持平 LLaMA-3-405B**
- **中文 + 长上下文**:V3 显著强(数据 + MLA 加成)
- **代码 / SWE-Bench**:V3 表现亮眼
- **多模态 / 工具 / "感觉"**:GPT-4o 仍有体验优势,LLaMA / V3 都在追赶

### 27.5.2 评测的局限

- benchmark 容易过拟合
- 真实 chat 体验需人工评测
- 不同测试方法分数可差 5+ 点

实际选型应结合**自家场景做 holdout 测试**。

## 27.6 工程哲学:效率派 vs 蛮力派

### 27.6.1 蛮力派(LLaMA-3)

- **思路**:简单架构 + 大数据 + 大算力,把已验证的"配方"推到极致
- **优势**:实现简单,工程稳定,生态容易复用
- **代价**:训练贵、推理也贵
- **代表问题**:Meta、Google(Gemini Ultra)、OpenAI(GPT-4 dense 时代)

### 27.6.2 效率派(DeepSeek-V3)

- **思路**:架构创新(MLA / MoE)+ 系统极致(FP8 / DualPipe)+ Inference-aware scaling
- **优势**:同质量,1/10 训练成本,推理也便宜
- **代价**:工程复杂度高、调参难度高、需要专业团队
- **代表**:DeepSeek、Mistral(MoE 系列)、部分国内团队

### 27.6.3 哪一条更难?

两条都很难,**只是难在不同地方**:

- 蛮力派难在**规模执行**(2 万卡协同 54 天不出错)
- 效率派难在**算法 + 系统全栈协同**(MLA + MoE + FP8 + DualPipe 任一环节崩了都不行)

### 27.6.4 趋势

2025 之后,**两条路线都在融合**:

- LLaMA 系列开始研究 MoE(LLaMA-4 据社区传闻有 MoE 设计)
- DeepSeek 也在尝试更激进的 dense 优化

但短期内,效率派的"美元产出比"明显占优。

## 27.7 各自最强场景

| 场景 | 优先选 |
| --- | --- |
| **本地小模型部署**(7B-32B) | LLaMA-3.x / Qwen / Mistral / DeepSeek 蒸馏 |
| **学术研究 / 复现** | LLaMA-3(架构简单,文档丰富) |

| **长上下文(64k+)** | **DeepSeek-V3**(MLA) |
| **中文场景** | **DeepSeek-V3 / Qwen** |
| **多模态** | LLaMA-3.2 Vision / Qwen-VL(V3 暂无多模态版) |

| **追求"最简 dense 训练 baseline"** | LLaMA-3 |

→ **没有银弹**。同代两家把开源生态推到了"按需选择"的成熟阶段。

## 27.8 开源生态影响力

### 27.8.1 LLaMA-3

- 模型卡 + 权重开源(商业可用,7 亿月活以下)
- 训练论文极详(*The Llama 3 Herd*)
- 衍生生态海量(精调 / 量化 / 多语言 / 多模态)

### 27.8.2 DeepSeek-V3

- 模型卡 + 权重开源(MIT 风格,无月活限制)
- 训练论文 + 部分 kernel(DeepEP、FlashMLA)开源
- 在中文社区影响力极大
- 国际开发者也大量复现

### 27.8.3 谁推动了"开源对标 GPT-4"?

2024 末-2025 初最常引用的"开源对标 GPT-4"模型是 **V3 + R1**。LLaMA-3-405B 是同期重要的 dense 标杆。

两者共同的功劳:**让人们相信开源能与闭源并肩**,从而显著改变行业 build-vs-buy 决策。

## 🎨 27.9.A 通俗类比合集

### 类比一: 美式肌肉车 vs 日式精益车 🚗

> LLaMA-3 = 美式肌肉车;V3 = 日式精益车。

- **肌肉车**:大引擎(dense),油门一脚到底,够暴力 → 简单可靠
- **精益车**:小排量涡轮 + 智能变速(MoE + MLA + FP8),同等动力**油耗一半**
- **驾驶体验**:两者都能上 200,但赛道效率截然不同
- **维护成本**:肌肉车零件大但通用;精益车设计精细但需要熟练技师(对应工程团队门槛)

### 类比二: 巨型超市 vs 精品便利店连锁 🏬

> LLaMA-3 = 一家巨型超市;V3 = 全国 256 家精品便利店 + 1 个中央配送。

- 巨型超市(dense):什么都卖,每个客人都逛一遍
- 便利店(MoE):每客只去最近的 8 家,**总 SKU 是超市的 ×3**,但每客只看到 5%
- 长保鲜(MLA):便利店把库存压成压缩包(潜变量),不占货架
- 24h 不停业(FP8 + DualPipe):用更轻的设备 + 流水补货,运营成本极低

### 类比三: 综合性大学 vs 多个专精学院联盟 🎓

> LLaMA-3 = 一所巨型综合大学;V3 = 256 学院联盟 + 1 个通识院。

- 综合大学:所有学生都要学所有课
- 学院联盟:学生按兴趣去 8 个相关学院 + 1 个通识院打底
- 教学效率:同等学生数,学院联盟人均成本低 ~10×
- 文凭质量:不一定差,可能在专精领域反而更好

### 三个类比对比

| 类比 | 强调什么 | 适合记忆什么 |
| --- | --- | --- |
| 🚗 汽车 | **动力 / 效率取舍** | dense vs MoE |
| 🏬 超市 vs 便利店 | **稀疏激活 + 压缩库存** | MoE + MLA |
| 🎓 大学 vs 学院联盟 | **专精分工** | DeepSeekMoE 哲学 |

**记忆口诀**:
> **"LLaMA = 大引擎硬怼;V3 = MoE+MLA+FP8 巧打;同效果,1/10 成本。"**

## 27.10 常见疑问

### Q1: V3 真的"开源版 GPT-4"吗?
**A**: 在多数学术 benchmark 上接近或持平 GPT-4o / GPT-4 turbo,中文与长上下文超过。**真实体验(尤其多模态、工具、风格)仍有差距**,但开源圈第一次到这个水位。

### Q2: 我选了 V3 后,工程能 hold 住吗?
**A**: 看团队。
- 用 API:**完全 hold 住**(DeepSeek 官方 API + 第三方)
- 自部署(vLLM / SGLang):**中等难度**,需要熟悉 MoE 量化与 EP
- 自训练:**极高门槛**,需要全栈团队

### Q3: 我是产品方,该等 LLaMA-4 还是用 V3?
**A**: 看时间表与场景。当前(2025-2026):
- 想立刻上线、追求成本 → V3 / Qwen
- 想生态稳定 / 跨语种通用 → LLaMA-3.x
- 等 LLaMA-4 → 不可知风险大

### Q4: V3 没多模态,我怎么办?
**A**: 用专门多模态模型(Qwen-VL / LLaVA / LLaMA-3.2 Vision),或等 DeepSeek 后续 V3-VL 系列。Agent 场景常组合多个模型分工。

### Q5: 我学到一身 MoE 知识,LLaMA-3 dense 不学了行吗?
**A**: 不行。Dense 仍是主流(尤其端侧、小模型、研究基线)。**两条路都要懂**,前 4 部分书的知识是地基,§23-26 的 MoE/MLA 是高级建筑。

## 27.11 本章小结

### 核心对比

```text
LLaMA-3-405B          DeepSeek-V3 671B
└── Dense + GQA       ├── MoE 256+1, Top-8
└── bf16              ├── MLA(KV 压 ~57×)
└── 15.6T token       ├── bf16 + FP8 矩阵乘
└── ~16000 H100       ├── DualPipe + EP
└── ~$60M+            └── ~$5.5M
```

### 一句话总结

> **LLaMA-3 与 DeepSeek-V3 是 2024 开源 LLM 的两面镜子:LLaMA 用"简单架构 + 大算力 + 大数据"把蛮力派推到极致,V3 用"MLA + MoE + FP8 + DualPipe"把效率派推到极致。两者在多数 benchmark 上并驾齐驱,但训练成本相差 ~10×、推理经济性差更大;选型按场景(中文 / 长上下文 / 多模态 / 端侧 / 团队工程能力)而非绝对"谁强"。**

## 27.13 实战练习

### 练习 1 (★)

### 练习 2 (★★)
