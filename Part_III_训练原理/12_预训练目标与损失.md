## 摘要

预训练目标决定了"在大量未标注文本上,模型到底学什么"。它的形式直接决定模型架构、能力侧重、推理时的可用性。本章梳理:

- **NTP / CLM**(GPT 派):因果建模,自回归生成,LLM 默认
- **MLM**(BERT 派):双向理解,表示学习强,不适合生成
- **FIM**(Code-LLM 必备):中间填空,代码补全的关键
- **MTP**(DeepSeek-V3 引入):一次预测多 token,辅助 loss,提升训练效率
- **Span Corruption / UL2**:T5 / PaLM 系列的混合范式
- 评估:PPL / BPB,以及为何 PPL 不等同于下游能力

读完你能解释 GPT 与 BERT 数学上的核心差异、为什么 Code-LLM 必须加 FIM、以及 MTP 为什么不只是"快一点"。

## 12.1 三大预训练范式:NTP / MLM / Encoder-Decoder

| 范式 | 架构 | 信息流 | 生成能力 | 代表 |
| --- | --- | --- | --- | --- |
| **NTP / CLM** | Decoder-Only | 单向(causal) | ✅ 强 | GPT / LLaMA / DeepSeek |
| **MLM** | Encoder-Only | 双向 | ❌ 弱(需额外解码) | BERT / RoBERTa |
| **Span / Denoise** | Encoder-Decoder | 双向 → 单向 | ✅ | T5 / BART |
| **PrefixLM / UL2** | 单模型混合 | 局部双向 + 后段单向 | ✅ | UL2 / PaLM-2 |

现代生成式 LLM 几乎一致选 **NTP**;BERT 派在分类、检索仍有用。

## 12.2 Next-Token Prediction(NTP / CLM)

### 12.2.1 形式

给定序列 $x_1, \ldots, x_T$,模型在每个位置预测下一个 token:

$$
\mathcal{L}_{\text{NTP}} = -\frac{1}{T-1} \sum_{t=1}^{T-1} \log p_\theta(x_{t+1} \mid x_{\leq t}) \tag{12.1}
$$

由 §02 已知:这等价于 cross-entropy = 经验分布到模型分布的 KL。

### 12.2.2 工程要点

- **Causal mask**(§07):保证位置 $t$ 只看 $\leq t$
- **Shift**:`logits[:, :-1]` 与 `tokens[:, 1:]` 对齐(§10.5.2)
- **Padding token 不计 loss**:`ignore_index = pad_id`
- **Loss 平均策略**:按 token 数平均(per-token CE),而非按样本

### 12.2.3 与"教师强迫"(Teacher Forcing)

NTP 训练时**总是用真实历史 $x_{\leq t}$**(不是模型自己生成),又称 teacher forcing。推理时改用模型自己生成的 token —— 这造成 **train-inference distribution shift**(暴露偏差),但实际损失有限,LLM 仍以此为主。

---

## 12.3 Masked Language Modeling(MLM,BERT 风格)

### 12.3.1 形式

随机掩盖序列中 15% 的 token,模型预测被掩盖位置:

$$
\mathcal{L}_{\text{MLM}} = -\frac{1}{|\mathcal{M}|} \sum_{t \in \mathcal{M}} \log p_\theta(x_t \mid x_{\setminus \mathcal{M}}) \tag{12.2}
$$

其中 $\mathcal{M}$ 是被掩盖位置集合,$x_{\setminus \mathcal{M}}$ 是未被掩盖的位置(双向可见)。

### 12.3.2 双向 vs 因果

MLM 用 **bi-directional attention**(无 causal mask):每个位置可看序列任意其他位置 → 强**表示**,但不适合直接生成("从左到右"的自回归与双向不兼容)。

### 12.3.3 80 / 10 / 10 规则

被选中的 15% token:

- 80% 替换为 `[MASK]`
- 10% 替换为**随机 token**
- 10% **保持原 token 不变**

目的:防止模型只学"识别 `[MASK]` → 预测"的捷径(实际下游 fine-tune 时输入没 `[MASK]`)。

### 12.3.4 局限

- 每 batch 只对 15% 位置算 loss → **样本效率低**
- 不适合生成
- 后续 RoBERTa、ELECTRA 等做了改进,但整体路线已被 NTP 派碾压

## 12.4 Fill-in-the-Middle(FIM / PSM / SPM)

### 12.4.1 动机

代码补全场景:已知**光标前的代码 + 光标后的代码**,补中间。纯 NTP 模型只会续写不会"补中间",对 IDE 集成不友好。

### 12.4.2 重排技巧

把一段 token 拆成 prefix / middle / suffix 三段,**重排顺序**后仍按 NTP 训练:

**PSM 格式**(Prefix-Suffix-Middle):

```text
<PRE> prefix <SUF> suffix <MID> middle <EOT>
```

**SPM 格式**(Suffix-Prefix-Middle):

```text
<PRE> <SUF> suffix <MID> prefix <MID> middle <EOT>
```

(细节见 Bavarian et al., 2022)

训练时损失只算 middle 部分;**架构不变,只是数据视角变了**。

### 12.4.3 比例

混合 50% NTP + 50% FIM 是 OpenAI 在 *FIM* 论文中推荐配方。Code-LLM 几乎全部采用 FIM,Star-Coder、DeepSeek-Coder、CodeLlama 都有自己版本的特殊 token 与变体。

### 12.4.4 不仅是代码

FIM 思路也可用于文本编辑、长文档插入等。但在通用 LLM 训练中,FIM 占比通常较小(< 10%)。

## 12.5 Multi-Token Prediction(MTP,DeepSeek-V3)

### 12.5.1 动机

NTP 只在最后一个隐藏向量上预测**下一个** token,信号稀疏。若同时预测**未来 N 个 token**,梯度信号更丰富 → 训练效率与最终质量都受益。

### 12.5.2 结构(DeepSeek-V3 风格)

主干 Transformer 输出后,**接 N 个串行的 MTP head**(每个本身是一个小 Transformer 层 + LM head),分别预测下一个 / 下下个 / ... / 下第 N 个 token:

$$
\mathcal{L}_{\text{MTP}} = \mathcal{L}_{\text{NTP}} + \lambda \cdot \frac{1}{N-1} \sum_{k=2}^{N} \text{CE}_k \tag{12.3}
$$

第 1 个 head 就是普通 NTP loss;后续 head 是辅助 loss,权重 $\lambda \sim 0.3$。

### 12.5.3 关键工程点

- MTP head **串行而非并行**:第 $k$ 个 head 接收第 $k-1$ 个的中间表示,保证因果性
- 推理时**可丢掉 MTP head**(只用主 head),不影响生成
- 推理时**也可保留并用于 speculative decoding**(§21):一次 forward 给多个候选 token

### 12.5.4 收益

DeepSeek-V3 报告:加 MTP 在同 token 数下 loss 显著下降;推理时 speculative decoding 接受率 ~85%,吞吐提升 ~1.8×。

## 12.6 Span Corruption / UL2 / Mixture-of-Denoisers

### 12.6.1 T5 的 Span Corruption

把输入中**随机连续 span** 替换为单一哨兵 token `<X>`,解码器要还原原 span。比 BERT 的单 token MLM 更高效。

### 12.6.2 UL2 的 Mixture-of-Denoisers

PaLM-2 等用三种 denoising 任务混合:

- **R-denoiser**: 短 span,15% 掩盖比 → 类 MLM
- **S-denoiser**: 前缀 LM(类 Prefix-LM)
- **X-denoiser**: 长 span,大比例掩盖 → 强迫长程理解

训练时按某权重混合,以"一个模型多能力"。

### 12.6.3 现状

学术上有效,工业上**主流仍是纯 NTP**(更简单、更可预测)。FIM 与 MTP 是 NTP 派的两个最重要扩展。

## 12.7 训练 / 验证 / 评估指标

### 12.7.1 训练 loss

主要看:

- **整体 CE / PPL** 是否下降
- 各**领域子集**的 loss(代码 / 中文 / 数学)是否同步下降
- **梯度范数**是否稳定(<10),否则警惕爆炸

### 12.7.2 Held-out 验证

每 N 步在验证集(同分布,但训练未见)上跑 CE 与 PPL。曲线应平稳下降。

### 12.7.3 下游评测

PPL 仅相关于"建模文本的能力",**不直接等于能力**。常用 benchmark:

- **常识 / 知识**:MMLU、AGIEval、CEval(中文)
- **推理 / 数学**:GSM8K、MATH、AIME
- **代码**:HumanEval、MBPP、SWE-Bench
- **长上下文**:Needle-In-A-Haystack、Long-Bench

### 12.7.4 跨 tokenizer 公平比较

→ 用 **BPB**(bits per byte),见 §02.5.5。

## 12.8 完整代码实现

### 12.8.1 NTP loss(已在 §10.5.2)

```python
loss = F.cross_entropy(logits[:, :-1].reshape(-1, V),
                      tokens [:, 1:].reshape(-1),
                      ignore_index=pad_id)
```

### 12.8.2 MLM 采样器

```python
import torch

def mlm_collate(tokens: torch.Tensor, mask_id: int, vocab_size: int,
                p: float = 0.15):
    """
    tokens: (B, T) long
    返回 (masked_inputs, labels) — labels 上未被 mask 的位置 = -100,不计 loss。
    """
    B, T = tokens.shape
    rand = torch.rand(B, T)
    chosen = rand < p                                     # 15% 选中
    masked = chosen & (torch.rand(B, T) < 0.8)            # 80% [MASK]
    rand_tok = chosen & ~masked & (torch.rand(B, T) < 0.5)  # 10% 随机
    # 剩 10% 保留原 token

    inputs = tokens.clone()
    inputs[masked]  = mask_id
    inputs[rand_tok] = torch.randint(0, vocab_size, (rand_tok.sum(),))
    labels = torch.where(chosen, tokens, torch.full_like(tokens, -100))
    return inputs, labels
```

### 12.8.3 FIM 重排

```python
def fim_rearrange(tokens: list[int], tok_pre: int, tok_mid: int,
                  tok_suf: int, tok_eot: int, rng=None):
    """PSM 格式: <PRE> prefix <SUF> suffix <MID> middle <EOT>"""
    import random
    rng = rng or random
    n = len(tokens)
    if n < 8: return tokens
    a, b = sorted(rng.sample(range(n), 2))
    prefix, middle, suffix = tokens[:a], tokens[a:b], tokens[b:]
    return [tok_pre, *prefix, tok_suf, *suffix, tok_mid, *middle, tok_eot]
```

### 12.8.4 MTP loss(简化)

```python
def mtp_loss(main_logits, mtp_logits_list, targets, pad_id, lam=0.3):
    """
    main_logits: (B, T, V) — NTP 主输出
    mtp_logits_list: [(B, T, V), ...] — k=2..N 的额外 head 输出,
                                       与 targets 错位 k-1
    targets: (B, T)
    """
    V = main_logits.size(-1)
    loss_main = F.cross_entropy(
        main_logits[:, :-1].reshape(-1, V),
        targets[:, 1:].reshape(-1),
        ignore_index=pad_id,
    )
    loss_aux = 0.0
    for k, mtp_logits in enumerate(mtp_logits_list, start=2):
        L = T - k
        loss_aux += F.cross_entropy(
            mtp_logits[:, :L].reshape(-1, V),
            targets[:, k-1: T-1].reshape(-1),  # 错位
            ignore_index=pad_id,
        )
    return loss_main + lam * loss_aux / max(len(mtp_logits_list), 1)
```

## 🎨 12.9.A 通俗类比合集

### 类比一: 完形填空 vs 接龙作文 ✍️

> 不同的练习题型练出不同的能力。

- **NTP** = **接龙作文**:老师只给你写一半,让你续写到结尾 → 练"流畅生成"
- **MLM** = **完形填空**:老师挖空 15% 词让你填 → 练"双向理解"
- **FIM** = **写中间段落**:给你"前情" + "结局",让你补中间剧情 → 练"代码补全"
- **MTP** = **跳棋一次走 4 步**:不光预测下一步,还预测下下、下下下步 → 信号更密集

### 类比二: 厨师培训 👨‍🍳

> 不同培训方式练出不同厨艺。

- **NTP** = 站在炉边看师傅做整道菜,然后从头开始模仿
- **MLM** = 师傅把一道菜中间几个步骤"打码",让徒弟猜师傅做了什么
- **FIM** = 师傅给"准备食材"和"装盘"两步,让徒弟补"炒制"过程
- **MTP** = 徒弟边做边预测"师傅下一步是炒、下下步是焯水、下下下步是出锅"
- **UL2** = 一周里既考完形填空、又考接龙、又考改写 — 啥都练点

### 类比三: 阅读测验题型 📖

> 大学英语四六级里的各种阅读题。

- **NTP** = **作文续写**:看一半文章接着写,练表达
- **MLM** = **阅读理解的填空题**:抠掉若干词让你填,练理解
- **Span Corruption** = **段落改写题**:整段抹掉,要求按上下文重写
- **MTP** = **预测接下来 3 段话各自讲什么**:不只是下一段,而是后面几段的纲要

### 三个类比对比

| 类比 | 强调什么 | 适合记忆什么 |
| --- | --- | --- |
| ✍️ 作文 | **生成 vs 填空** | NTP / MLM / FIM 区别 |
| 👨‍🍳 厨师 | **训练信号密度** | MTP 多 head 的意义 |
| 📖 测验 | **题型多样化** | UL2 mixture-of-denoisers |

**记忆口诀**:
> **"NTP 写下一字,MLM 填漏字,FIM 补中段,MTP 预测一串。"**

## 12.10 常见疑问

### Q1: 为什么不用 next sentence prediction(NSP)?
**A**: BERT 原版有 NSP(两段是否相邻),后续研究(RoBERTa)发现 NSP 几乎无用,**还干扰主 MLM 信号**。LLM 时代彻底不用辅助任务,信任纯 NTP 的统一性。

### Q2: NTP 不能学到双向理解吗?
**A**: 能,但隐含。NTP 在每个位置都"用左侧上下文预测右侧"——大量样本下,各种语言学关系都能被捕捉。实证显示纯 NTP 在理解任务上已可超过 BERT(GPT-3 时代的发现)。

### Q3: 训练 FIM 后,模型 NTP 能力会退化吗?
**A**: 几乎不会。FIM 的本质是同样的 NTP 目标,只是 token 序列被重排。OpenAI 的论文显示 50% NTP + 50% FIM 在 NTP 评测上无显著退化。

### Q4: MTP 与 speculative decoding 什么关系?
**A**: MTP head 在推理时**可以**用作 speculative decoding 的 draft generator(给候选)。本节的"训练目标"与 §21 的"推理优化"在 DeepSeek-V3 是同一套权重的两种用法。

### Q5: 同一模型能同时学 NTP + MLM 吗?
**A**: 可以,但效果通常不如专心一种。GLM(清华)用一种混合形式;UL2 也尝试过。工程上少见。

## 12.11 本章小结

### 核心公式

**NTP**:

$$
\mathcal{L} = -\frac{1}{T-1} \sum_t \log p_\theta(x_{t+1} \mid x_{\leq t})
$$

**MLM**:

$$
\mathcal{L} = -\frac{1}{|\mathcal{M}|} \sum_{t\in\mathcal{M}} \log p_\theta(x_t \mid x_{\setminus \mathcal{M}})
$$

**MTP**:

$$
\mathcal{L} = \mathcal{L}_\text{NTP} + \lambda \cdot \tfrac{1}{N-1}\sum_{k=2}^{N} \text{CE}_k
$$

### 核心要点

| 要点 | 关键 |
| --- | --- |
| **NTP** | 现代生成式 LLM 默认 |
| **MLM** | 双向理解,样本效率低 |
| **FIM** | 同 NTP 但 token 重排,代码补全必备 |
| **MTP** | 多 head 多步预测,提升训练效率 |
| **评估** | PPL 不等同能力,看 benchmark |

### 一句话总结

> **现代生成式 LLM 的预训练目标几乎一致是 NTP(逐 token 交叉熵);FIM 通过重排 token 让模型学会"中间补全",MTP 通过额外 head 一次预测多步以加密训练信号——三者共享同一套架构与 cross-entropy 数学,只是看问题的角度不同。**

### 练习 3 (★★)

### 练习 4 (★★★)

实现简化版 MTP:在 GPTMini 输出后加 N=4 个 head(各一个 nn.Linear),计算公式 12.3 的 loss。对比纯 NTP 训练曲线。

### 练习 5 (★★★)

读 DeepSeek-V3 技术报告 §2.1,梳理它的 MTP head 串行结构与 inference 解耦机制,用伪代码画出 forward 流程。
