## 摘要

把 N 个 Transformer Block 堆叠起来不是一件平凡的事:激活值会被反复线性变换 + 非线性激活,数值容易爆炸或衰减,梯度难以稳定流回。**残差连接**(Residual)与**归一化**(Norm)是让深堆叠可训练的两个核心粘合机制:

- 残差: 让梯度有一条"高速公路"绕过非线性,缓解梯度消失
- 归一化: 在每层入口或出口把激活拉回稳定的尺度,避免方差漂移

本章系统比较 LayerNorm 与 RMSNorm 的代数与实证差异、Pre-Norm 与 Post-Norm 的优化曲面差异,以及现代 LLM(LLaMA、DeepSeek、Qwen…)几乎一致采用 **Pre-RMSNorm + 残差**的工程原因。

## 9.1 为什么深网络需要归一化与残差?

### 9.1.1 堆叠的两大病

假设把 $N$ 个非线性层 $f_l$ 简单堆叠 $h_N = f_N \circ \cdots \circ f_1(h_0)$:

1. **激活漂移**:每层做线性变换 + 非线性,激活的均值/方差容易随层数指数发散或衰减
2. **梯度消失/爆炸**:反向传播时梯度连乘 $\prod \partial f_l / \partial h_{l-1}$,稍微偏离 1 就是几十层后的极端值

ResNet(He et al., 2015)与 BatchNorm(Ioffe & Szegedy, 2015)在 CNN 上首次解决了这一问题。Transformer 的 LayerNorm + 残差是其在序列模型上的对应方案。

### 9.1.2 工程铁律

> **没有残差,堆 6 层就过拟合/发散;有残差,堆 100 层都能稳定训练。**
> **没有归一化,Attention 的 softmax 容易把激活推到饱和;有归一化,可在 fp16/bf16 下安全堆深。**

## 9.2 残差连接(Residual)

### 9.2.1 定义

子层 $f$ 加上恒等捷径(skip connection):

$$
h_{\text{out}} = x + f(x) \tag{9.1}
$$

Transformer Block 的两个子层(Attention、FFN)各自被这样包裹。

### 9.2.2 缓解梯度消失的直觉

对 $h_{\text{out}} = x + f(x)$ 求梯度:

$$
\frac{\partial h_{\text{out}}}{\partial x} = I + \frac{\partial f}{\partial x}
$$

即使 $\partial f / \partial x \approx 0$(子层学不到),恒等项 $I$ 仍把梯度**完整地传回上游**。这等价于给反向传播开了一条"高速公路"。

### 9.2.3 工程含义

- 残差**几乎零计算开销**,只是一次加法
- 它让模型可以"按需关闭"某个子层(把 $f$ 学成 0),给了优化器更大空间
- 对 LLM 解释性研究来说,**Residual Stream(残差流)** 成为分析中间表示的核心概念

## 9.3 LayerNorm:Transformer 原版默认

### 9.3.1 定义

Ba et al., 2016。对**单个样本的特征维度**做归一化:

给定 $x \in \mathbb{R}^d$,

$$
\mu = \frac{1}{d}\sum_{i=1}^{d} x_i, \quad \sigma^2 = \frac{1}{d}\sum_{i=1}^{d}(x_i - \mu)^2 \tag{9.2a}
$$

$$
\text{LN}(x) = \gamma \odot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta \tag{9.2b}
$$

其中 $\gamma, \beta \in \mathbb{R}^d$ 是可学习的缩放与偏置,$\epsilon$ 是数值稳定常数(通常 $10^{-5}$)。

### 9.3.2 关键性质

- 沿**特征维**而非"批"维做统计 → batch size 不影响,适合序列建模
- 推理与训练行为完全一致(不需要像 BN 那样保存 running stats)
- 把每个 token 的特征拉到零均值、单位方差,激活分布稳定

### 9.3.3 计算开销

每次需要算均值 + 方差 + 平方根 + 减法 + 除法,但相比矩阵乘法(O(d²) 量级)只是 $O(d)$,占比可忽略。

---

## 9.4 RMSNorm:现代 LLM 的默认

### 9.4.1 定义

Zhang & Sennrich, 2019。**只做方差归一化,不去均值**:

$$
\text{RMS}(x) = \sqrt{\frac{1}{d}\sum_{i=1}^{d} x_i^2 + \epsilon} \tag{9.3}
$$

$$
\text{RMSNorm}(x) = \gamma \odot \frac{x}{\text{RMS}(x)} \tag{9.4}
$$

注意:**没有 $\beta$,也不减 $\mu$**。

### 9.4.2 相对 LayerNorm 省了什么

| 项 | LayerNorm | RMSNorm |
| --- | --- | --- |
| 减均值 | ✅ | ❌ |
| 除方差 | ✅ | ❌(改除 RMS) |
| 学 $\beta$ | ✅ | ❌ |
| 计算量 | 完整 | 约省 30-50% |

经验上,**去掉均值化几乎不损失模型质量,但更快、更省显存、数值更稳**。LLaMA、Mistral、DeepSeek、Qwen、Gemma 全部采用 RMSNorm。

### 9.4.3 为什么去均值无害?

直觉:LLM 的激活分布在大量训练之后**已经接近零均值**,LayerNorm 的"减均值"步骤大多在做"接近 0 的减法",冗余。RMSNorm 直接信任这一点,省掉计算。

理论上,RMSNorm 对"等比缩放"具有不变性,而 LayerNorm 对"平移 + 等比缩放"具有不变性——但后者的平移不变性在 Transformer 中并不重要。

## 9.5 Pre-Norm vs Post-Norm

### 9.5.1 两种放置位置

**Post-Norm(原版 Transformer)**:

$$
h_{\text{out}} = \text{Norm}\big(x + f(x)\big) \tag{9.5}
$$

**Pre-Norm(GPT-2 以来主流)**:

$$
h_{\text{out}} = x + f\big(\text{Norm}(x)\big) \tag{9.6}
$$

### 9.5.2 训练稳定性差异

- **Post-Norm**:残差路径上叠加了 Norm → 梯度被反复缩放,深堆叠时容易出现梯度爆炸,需要复杂的 warmup + 小心调参
- **Pre-Norm**:残差路径是**纯加法**,梯度直通 → 极深堆叠仍稳定;代价是**输出未归一化**,需要在最后一层加一次 Final Norm

实证上,Pre-Norm 在百层级 Transformer 上几乎不需调参就能稳定训练。GPT-2、GPT-3、LLaMA 系列、DeepSeek 全部用 Pre-Norm。

### 9.5.3 一个常被忽视的代价

Pre-Norm 把每个子层的输入归一化后,**主干 residual stream 的方差会随层数缓慢累积**——因为 $\text{Var}(x + f(\text{Norm}(x))) \geq \text{Var}(x)$。这导致深层模型最后一层激活范数显著大于输入。Final RMSNorm 在收尾抹平这件事。

### 9.5.4 Sandwich-Norm 等变体

为兼顾两者,有方案在每个子层**前后都加 Norm**(Sandwich,如 GLM 用过),但额外开销与稳定性增益的性价比争议较大,主流仍是 Pre-Norm。

## 9.6 DeepNorm 等深网络稳定化技巧

### 9.6.1 DeepNorm

Wang et al., 2022。在残差里加一个固定缩放因子 $\alpha$:

$$
h_{\text{out}} = \text{Norm}\big(\alpha \cdot x + f(x)\big) \tag{9.7}
$$

并配合初始化时把 $W$ 缩 $\beta$ 倍。理论上证明可稳定训练 1000 层 Post-Norm Transformer。但工程上多数团队仍偏好"Pre-Norm + RMSNorm",DeepNorm 更多见于极深超大模型(MoE)实验。

### 9.6.2 QKNorm / Cosine attention

把 Q 和 K 在做点积前各自归一化:

$$
A_{ij} = \frac{\text{Norm}(Q_i) \cdot \text{Norm}(K_j)^\top}{\sqrt{d_k}} \cdot \tau
$$

避免 logits 过大、softmax 饱和。在大规模训练(GPT-4 类、Gemini、部分 MoE)中广泛使用。

### 9.6.3 Tanh-softmax 等 logit 裁剪

DeepSeek-V3 等模型在 attention logits 进入 softmax 前做 `30 * tanh(logit / 30)` 形式的软裁剪,进一步稳定数值。

## 9.7 完整代码实现

### 9.7.1 LayerNorm(显式)

```python
import torch
import torch.nn as nn

class LayerNorm(nn.Module):
    def __init__(self, d: int, eps: float = 1e-5):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d))
        self.beta  = nn.Parameter(torch.zeros(d))
        self.eps = eps

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # x: [..., d]
        mean = x.mean(dim=-1, keepdim=True)
        var  = x.var(dim=-1, keepdim=True, unbiased=False)
        x_hat = (x - mean) / torch.sqrt(var + self.eps)
        return self.gamma * x_hat + self.beta
```

### 9.7.2 RMSNorm(LLaMA 风格)

```python
class RMSNorm(nn.Module):
    def __init__(self, d: int, eps: float = 1e-6):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d))
        self.eps = eps

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # 用 fp32 计算均方根以稳数值,再 cast 回原 dtype
        in_dtype = x.dtype
        x32 = x.to(torch.float32)
        rms = x32.pow(2).mean(dim=-1, keepdim=True).add(self.eps).rsqrt()
        return (self.gamma * (x32 * rms)).to(in_dtype)
```

### 9.7.3 Pre-Norm Block 骨架

```python
class PreNormBlock(nn.Module):
    def __init__(self, d_model, n_heads, d_ff):
        super().__init__()
        self.norm1 = RMSNorm(d_model)
        self.attn  = MultiHeadAttention(d_model, n_heads)   # 来自 §07
        self.norm2 = RMSNorm(d_model)
        self.ffn   = SwiGLUFFN(d_model, d_ff)               # 来自 §08

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        x = x + self.attn(self.norm1(x))
        x = x + self.ffn (self.norm2(x))
        return x
```

**关键**:Norm 在子层入口、加法在子层出口,残差是**主干**。

## 9.8 复杂度与数值精度

### 9.8.1 复杂度

| 操作 | 复杂度 |
| --- | --- |
| LayerNorm/RMSNorm | $O(n \cdot d)$ |
| Attention | $O(n^2 d)$ |
| FFN | $O(n d^2)$ |

归一化对总 FLOPs 贡献 ≪ 1%,但**显存访问密集**,在 GPU 上需要专门的 fused kernel(FlashAttention、Apex、xFormers 都有)。

### 9.8.2 混合精度训练的陷阱

- bf16/fp16 训练时,**RMS 的平方和**很容易 overflow(尤其大 hidden 时)。标准做法是**在 fp32 里计算 RMS**(见 §9.7.2 代码)
- $\epsilon$ 选 $10^{-6}$ 通常够;过大会平滑掉小信号,过小会数值爆炸
- $\gamma$ 初始化为 1,$\beta$(若有)初始化为 0 —— 等价于"什么都不做",便于梯度健康启动

## 9.9 与 BatchNorm / GroupNorm 的对比

| 归一化 | 沿哪个维度归一 | 依赖 batch? | 用于 LLM? |
| --- | --- | --- | --- |
| **BatchNorm** | (B) | ✅ 强(running stats) | ❌ 不适合(变长序列、小 batch) |
| **InstanceNorm** | (H, W) 单样本 | ❌ | 视觉常见 |
| **GroupNorm** | 通道分组 | ❌ | 视觉常见 |

**核心要点**:序列模型用 batch-independent 的归一化,**RMSNorm 是当前事实标准**。

## 🎨 9.9.A 通俗类比合集

### 类比一: 高速公路 + 限速标牌 🛣️

> 残差 = 高速公路;归一化 = 每隔几公里的"限速标牌"。

- **残差(高速公路)**:即使旁边的辅路堵车(子层学得差),主路也不堵 → 梯度永远能传回
- **归一化(限速)**:防止某段路出现"飙车"(激活爆炸)或"龟速"(激活消失)
- **Pre-Norm**:**进辅路前先看限速**(每个子层入口先归一化)→ 主路一直畅通
- **Post-Norm**:**辅路回主路前看限速**(子层出口后归一化)→ 主路上每隔一段就有限速龙门架,容易追尾

### 类比二: 厨房里的"标准化半成品" 🍳

> 想象 32 道工序的中央厨房:每道工序前都要把"半成品"称重 + 调整温度,才能交给下一道。

- **LayerNorm**:称重(去均值) + 调温(除方差) + 加调味料($\gamma, \beta$)
- **RMSNorm**:**信任厨师的称重已经准**(不再去均值),只调温 + 加味精($\gamma$)→ 更快、更省事
- **残差**:每道工序旁边都备一份**原始食材**,如果厨师这道工序炒坏了,顾客至少还能吃到原料 → 信息不丢

### 类比三: 校园里的"标准化校服 + 升旗仪式" 🏫

> 学校怕学生千奇百怪,每周升旗后大家穿校服(归一化);但仍允许每个学生**保留自己的风格小物**(残差):

- **统一校服(归一化)**:大家身高体型在视觉上拉回一致,老师好讲课
- **保留自己物品(残差)**:防止"标准化"把每个学生独特性磨平
- **Pre-Norm**:每节课前先让学生换好校服 → 上课秩序好
- **Post-Norm**:每节课结束前才统一校服 → 课上可能已经吵翻天

### 三个类比对比

| 类比 | 强调什么 | 适合记忆什么 |
| --- | --- | --- |
| 🛣️ 高速公路 | **梯度通路** | 残差为什么必不可少 |
| 🍳 中央厨房 | **逐层标准化** | LN vs RMSNorm 差别 |
| 🏫 校服+物品 | **统一与个性的平衡** | Norm 与 Residual 的互补 |

**记忆口诀**:
> **"加法是高速,归一是限速,Pre 比 Post 稳。"**

## 9.10 常见疑问

### Q1: 为什么不在每层之后都做一次归一化和残差?
**A**: 实际上正是这样做的——每个 Transformer Block 内有两个子层(Attention、FFN),每个子层都包了一层 Norm + Residual。只是 Norm 位置(Pre/Post)的选择不同。

### Q2: Pre-Norm 的最终输出未归一化,会出问题吗?
**A**: 会。这就是为什么 Pre-Norm 模型的最后,在 LM head 之前再加一个 Final RMSNorm。LLaMA、DeepSeek 等都有这一层。

### Q3: $\gamma, \beta$ 可不可以全去掉?
**A**: 可以去 $\beta$(RMSNorm 这么做),但去 $\gamma$ 会损失"逐通道增益"的灵活性,效果显著变差。$\gamma$ 是 RMSNorm 的最后一份可学习参数。

### Q4: 残差直接相加,$x$ 与 $f(x)$ 量级差太多怎么办?
**A**: 这就是为什么必须先归一化:把 $x$ 的尺度控制住,$f$(Linear + 激活)输出的尺度也通过 $\gamma$ 控制,两者量级匹配。否则训练初期残差路径会被淹没或淹没 $f$。

### Q5: bf16 训练时为什么 RMS 要用 fp32 算?
**A**: bf16 只有 8 位尾数,平方再求和容易丢精度甚至 overflow($x^2$ 的范围比 $x$ 大平方倍)。标准做法是上 cast 到 fp32 做 RMS,再 cast 回 bf16 输出。代码见 §9.7.2。

## 9.11 本章小结

### 核心公式

**残差**:

$$
h_{\text{out}} = x + f(x)
$$

**LayerNorm**:

$$
\text{LN}(x) = \gamma \odot \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta
$$

**RMSNorm**:

$$
\text{RMSNorm}(x) = \gamma \odot \frac{x}{\sqrt{\tfrac{1}{d}\sum x_i^2 + \epsilon}}
$$

**Pre-Norm Block**:

$$
x \leftarrow x + \text{Attn}(\text{Norm}(x)) \,;\quad x \leftarrow x + \text{FFN}(\text{Norm}(x))
$$

### 核心要点

| 要点 | 关键 |
| --- | --- |
| **残差** | 给梯度开高速公路 |
| **LayerNorm** | 减均值 + 除方差 + 仿射 |
| **RMSNorm** | 只除 RMS,更快更稳 |
| **Pre vs Post** | Pre 稳定可深堆叠 |
| **混合精度** | RMS 用 fp32 算 |

### 一句话总结

> **残差给梯度开了一条恒等高速公路,归一化在每个子层入口把激活拉回稳定尺度;现代 LLM 的事实标准是"Pre-RMSNorm + 残差",凭这两件简单工具就能把深度堆到 100 层以上,稳定训练 bf16 大模型。**

在 §9.7.1 的 LayerNorm 之外,实现一个"无 $\beta$ 的 LayerNorm",训练一个 6 层小 Transformer,对比 loss 曲线——直观感受 $\beta$ 的作用(或没作用)。

在 RMSNorm 上去掉 $\gamma$(把它固定为 1),观察训练曲线如何变差。

### 练习 3 (★★)

用同一份数据训练两个 12 层 Transformer,一个 Pre-Norm 一个 Post-Norm,绘制 loss 曲线与梯度范数曲线,验证 §9.5.2 的稳定性差异。

### 练习 4 (★★)

在 bf16 训练中故意把 RMSNorm 改成全程 bf16(不上 cast),观察 loss 何时崩溃;再加回 fp32 计算,验证修复。

### 练习 5 (★★★)

读 LLaMA-3 / DeepSeek-V3 源码,梳理出**所有 Norm 层的位置**(Attention 前、FFN 前、Block 后、Final、Logit 前 …),绘制完整数据流图。
