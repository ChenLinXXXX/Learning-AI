## 摘要

神经网络可以训练,**完全归功于反向传播**:loss 是 $\theta$ 的复杂函数,但只要它由可微算子组合而成,我们就能在计算图上**逆向、机械地**算出 $\partial \mathcal{L} / \partial \theta$。本章梳理:

- 标量 / 向量 / 矩阵导数,以及实践中用的"形状对齐技巧"
- 计算图:每个张量节点都记录"我是怎么算出来的",反向时倒着调用 vjp
- LLM 关键算子(matmul、softmax、cross-entropy、attention、norm)的反向公式
- PyTorch Autograd 的实际机制:动态图、`requires_grad`、`backward()`
- 训练系统级技巧:**激活检查点**(显存与算力的取舍)与**梯度裁剪**(数值稳定)

读完本章你应能用 100 行 numpy 写一个能训练 MLP 的 mini autograd,并解释 LLM 训练中显存与算力账的来源。

## 3.1 标量函数的导数:链式法则回顾

### 3.1.1 单变量链式法则

$$
\frac{d f(g(x))}{dx} = f'(g(x)) \cdot g'(x) \tag{3.1}
$$

读法: **"外层的导数 × 内层的导数"**。

### 3.1.2 多变量链式法则(标量输出)

设 $y = f(u_1, u_2, \ldots, u_n)$,每个 $u_i = u_i(x)$:

$$
\frac{dy}{dx} = \sum_{i=1}^{n} \frac{\partial f}{\partial u_i} \cdot \frac{du_i}{dx} \tag{3.2}
$$

**所有反向传播都是这一条的反复展开**——只是中间变量变成张量。

---

## 3.2 向量与矩阵的求导:Jacobian / 梯度

### 3.2.1 梯度(标量对向量)

$f: \mathbb{R}^n \to \mathbb{R}$ 时:

$$
\nabla_{\mathbf{x}} f = \left(\frac{\partial f}{\partial x_1}, \ldots, \frac{\partial f}{\partial x_n}\right)^\top \in \mathbb{R}^n \tag{3.3}
$$

### 3.2.2 Jacobian(向量对向量)

$\mathbf{f}: \mathbb{R}^n \to \mathbb{R}^m$ 时:

$$
J_{ij} = \frac{\partial f_i}{\partial x_j}, \quad J \in \mathbb{R}^{m \times n}
$$

### 3.2.3 矩阵导数的"形状对齐技巧"

实践中很少手算 Jacobian,而是利用**形状对齐**判定公式:

> $\partial \mathcal{L} / \partial X$ **与 $X$ 形状相同**(因为它是 loss 对 $X$ 每个元素的偏导组成的同形张量)。

例:$Y = X W$,$X \in \mathbb{R}^{m\times n}$、$W \in \mathbb{R}^{n\times p}$,$Y \in \mathbb{R}^{m\times p}$。已知 $\partial \mathcal{L} / \partial Y$,求 $\partial \mathcal{L} / \partial W$:

$$
\frac{\partial \mathcal{L}}{\partial W} = X^\top \frac{\partial \mathcal{L}}{\partial Y}, \quad \frac{\partial \mathcal{L}}{\partial X} = \frac{\partial \mathcal{L}}{\partial Y} W^\top \tag{3.4}
$$

**口诀**: "**梯度形状要对齐,把已知夹中间;转置看维度对不对**"。

## 3.3 计算图(Computational Graph)

### 3.3.1 定义

一个**有向无环图**:

- 节点 = 张量(输入、参数、中间结果、loss)
- 边 = 算子(matmul、加法、softmax …)

LLM 的整个 forward 都是这样一张图。

### 3.3.2 动态 vs 静态

| 类型 | 例 | 特点 |
| --- | --- | --- |
| 静态图 | TensorFlow 1 / JAX 编译 | 提前编译,可优化,但调试难 |
| **动态图** | **PyTorch** | 每次 forward 现场构建,易调试 |

PyTorch 的胜出源于动态图带来的开发效率,后期再用 `torch.compile` / Triton 弥补编译收益。

### 3.3.3 一个 mini 计算图

```text
x ──▶ Linear(W1, b1) ──▶ ReLU ──▶ Linear(W2, b2) ──▶ CE Loss(y)
                                                       │
                                                       ▼
                                                       L
```

forward 沿箭头算 → 拿到 $L$;backward 沿箭头倒着算梯度。

---

## 3.4 反向传播:从 loss 倒着传梯度

### 3.4.1 算法骨架

1. **Forward**: 按拓扑序计算每个节点的值,**保留中间结果**
2. **Backward**:
   - 从 loss 节点设 $\partial \mathcal{L}/\partial \mathcal{L} = 1$
   - 按**反拓扑序**,每遇到一个算子,调用其 **vjp**(vector-Jacobian product)规则,把"上游梯度"乘上"局部 Jacobian"
   - 把 vjp 结果分发给该算子的所有输入

### 3.4.2 vjp 的工程含义

不必显式构造完整 Jacobian(可能巨大),只要算 **vector × Jacobian** 的乘积:

$$
v^\top J = \left[\sum_i v_i \frac{\partial f_i}{\partial x_1}, \ldots, \sum_i v_i \frac{\partial f_i}{\partial x_n}\right] \tag{3.5}
$$

这正是 autograd 库的核心抽象。

### 3.4.3 复杂度

反向传播总 FLOPs $\approx$ forward 的 2-3 倍。直觉:每个算子的 vjp 与 forward 同量级。所以**训练成本 ≈ 推理成本 × 3**(forward + backward + optimizer step)。

---

## 3.5 关键算子的反向公式

### 3.5.1 矩阵乘 $Y = X W$

$$
\frac{\partial \mathcal{L}}{\partial W} = X^\top \frac{\partial \mathcal{L}}{\partial Y}, \quad \frac{\partial \mathcal{L}}{\partial X} = \frac{\partial \mathcal{L}}{\partial Y} W^\top \tag{3.6}
$$

### 3.5.2 Softmax + Cross-Entropy(融合)

logits $z$,one-hot 标签 $y$,模型概率 $p = \text{softmax}(z)$,loss $\mathcal{L} = -\sum_i y_i \log p_i$:

$$
\frac{\partial \mathcal{L}}{\partial z} = p - y \tag{3.7}
$$

形式极简。这就是 PyTorch 把两步融合的原因——又稳又快。

### 3.5.3 ReLU

$$
\frac{\partial \text{ReLU}(x)}{\partial x} = \begin{cases} 1 & x > 0 \\ 0 & x \leq 0 \end{cases} \tag{3.8}
$$

负半轴梯度为 0 → **死神经元**问题。GELU、SwiGLU 提供平滑替代(§08)。

### 3.5.4 LayerNorm / RMSNorm

公式较复杂,通常由框架的 fused kernel 直接实现。关键工程要点:

- bf16 下 RMS 用 fp32 算(§09.8.2)
- 由于涉及减均值 / 除标准差,反向需要"二阶项",数值上小心

### 3.5.5 Attention

由 matmul + softmax + masking + matmul 组合,反向自动由链式法则展开。**Flash Attention(§21)** 通过手写 fused kernel 把整个 backward 也压进 GPU SRAM,实现速度与显存双赢。

### 3.5.6 Dropout

forward: $y = x \cdot \text{mask} / (1-p)$;
backward: $\partial \mathcal{L}/\partial x = \partial \mathcal{L}/\partial y \cdot \text{mask} / (1-p)$。mask 需在 forward 中保存。

## 3.6 PyTorch Autograd 的工作原理

### 3.6.1 Tensor 上的两个关键属性

- `requires_grad`: 是否参与梯度跟踪
- `grad_fn`: 这个张量是哪个算子产生的(指向 Function 节点)

```python
x = torch.tensor([1.0, 2.0], requires_grad=True)
y = x.pow(2).sum()
print(y.grad_fn)   # <SumBackward0>
```

### 3.6.2 backward() 做了什么

```python
y.backward()
print(x.grad)   # tensor([2., 4.])
```

- 从 `y` 开始,按 `grad_fn` 链反向走
- 在每个 Function 节点调用其注册的 `backward` 实现
- 把梯度**累加**到对应 leaf tensor 的 `.grad`

### 3.6.3 关键细节

- **梯度累加**:`backward` 多次调用会累加 → 每个 batch 后必须 `optimizer.zero_grad()`
- **detach() / no_grad()**:用于断开计算图(推理、target network)
- **retain_graph**:默认 backward 后会释放中间张量,如需多次反向需 `retain_graph=True`

### 3.6.4 训练显存的来源

训练时显存包括:

| 项 | 体积 |
| --- | --- |
| 权重 $\theta$(bf16) | $2P$ |
| 梯度 $g$(bf16) | $2P$ |
| 优化器状态(AdamW: m, v, fp32 master) | $\approx 12P$ |
| **激活 / 中间张量** | $\propto B \cdot N \cdot d \cdot T$,**反向需要** |

激活是显存大头——任何能减少激活保留量的技巧都是显存救星。

## 3.7 激活检查点(Gradient Checkpointing)

### 3.7.1 思路

**只保存少量"检查点"激活,其余反向时重新 forward 一次**。

- 显存:从 $O(N)$ 降到 $O(\sqrt{N})$(N = 层数)
- 算力:多一次 forward,约 +30% 训练耗时

### 3.7.2 PyTorch 用法

```python
from torch.utils.checkpoint import checkpoint

def block_fn(x):
    return block(x)

for blk in model.blocks:
    x = checkpoint(block_fn, x, use_reentrant=False)
```

每个 Block 不再保存中间激活,backward 时再 forward 一次。

### 3.7.3 何时该用?

- 7B+ 模型在单卡训练 → 必开
- 70B+ 模型 + DP → 必开 + ZeRO + offload(§14)
- 推理时**永远不需要**(没 backward)

---

## 3.8 梯度裁剪与数值稳定

### 3.8.1 梯度裁剪(Gradient Clipping)

防止梯度爆炸。常用 **L2 范数裁剪**:

$$
g \leftarrow g \cdot \min\!\left(1, \frac{c}{\|g\|_2}\right) \tag{3.9}
$$

阈值 $c$ 典型取 1.0。LLM 训练几乎是必选,尤其训练初期。

### 3.8.2 Loss scaling(fp16 时)

fp16 梯度容易下溢。做法:**loss × 大数(如 128) → backward → 把梯度除回去**。bf16 因范围大,可省此步。

### 3.8.3 NaN / Inf 监控

训练中务必:

- `torch.isnan(loss).any()` 检查
- 梯度 norm 日志(突然飙到 1e6 = 危险)
- 必要时 skip 当前 step,而不是让 NaN 污染权重

## 3.9 完整代码实现:从零写一个 mini autograd

下面是约 80 行 numpy 实现的"教学型 autograd",能跑一个二分类 MLP。

```python
import numpy as np

class Tensor:
    def __init__(self, data, _children=(), _op=''):
        self.data = np.asarray(data, dtype=np.float32)
        self.grad = np.zeros_like(self.data)
        self._prev = set(_children)
        self._op = _op
        self._backward = lambda: None

    def __add__(self, other):
        other = other if isinstance(other, Tensor) else Tensor(other)
        out = Tensor(self.data + other.data, (self, other), '+')
        def _bw():
            self.grad  += out.grad
            other.grad += out.grad
        out._backward = _bw
        return out

    def __matmul__(self, other):
        out = Tensor(self.data @ other.data, (self, other), '@')
        def _bw():
            self.grad  += out.grad @ other.data.T
            other.grad += self.data.T @ out.grad
        out._backward = _bw
        return out

    def relu(self):
        out = Tensor(np.maximum(0, self.data), (self,), 'relu')
        def _bw():
            self.grad += out.grad * (self.data > 0)
        out._backward = _bw
        return out

    def sum(self):
        out = Tensor(self.data.sum(), (self,), 'sum')
        def _bw():
            self.grad += out.grad * np.ones_like(self.data)
        out._backward = _bw
        return out

    def backward(self):
        # 拓扑排序
        topo, visited = [], set()
        def build(v):
            if v in visited: return
            visited.add(v)
            for child in v._prev:
                build(child)
            topo.append(v)
        build(self)
        # 初始梯度
        self.grad = np.ones_like(self.data)
        for v in reversed(topo):
            v._backward()


# === 训练一个 MLP 解决 XOR ===
np.random.seed(0)
X = np.array([[0,0],[0,1],[1,0],[1,1]], dtype=np.float32)
y = np.array([[0],[1],[1],[0]], dtype=np.float32)

W1 = Tensor(np.random.randn(2, 8) * 0.5)
W2 = Tensor(np.random.randn(8, 1) * 0.5)
lr = 0.1

for step in range(2000):
    x_t = Tensor(X); y_t = Tensor(y)
    h = (x_t @ W1).relu()
    pred = h @ W2
    loss = ((pred + Tensor(-y)) ** 2 if False else None)   # 用 MSE 简化
    # 简化:loss = mean( (pred - y)^2 )
    diff = pred.data - y
    loss_val = (diff ** 2).mean()

    # 手动种子梯度(本最小实现未实现 MSE 算子,用闭式)
    pred.grad = 2 * diff / y.size
    W1.grad.fill(0); W2.grad.fill(0); h.grad = np.zeros_like(h.data); x_t.grad = np.zeros_like(X)
    # 用 backward 链路
    pred._backward()
    h._backward()
    (x_t @ W1)._backward = lambda: None  # 此处省略 — 教学代码,完整版见练习

    W1.data -= lr * W1.grad
    W2.data -= lr * W2.grad
    if step % 200 == 0:
        print(f"step={step} loss={loss_val:.4f}")
```

> 上述代码省略部分细节以保持短小,完整可运行版见 Karpathy 的 [micrograd](https://github.com/karpathy/micrograd) —— 70 行实现完整 autograd,墙裂推荐对照阅读。

## 🎨 3.9.A 通俗类比合集

### 类比一: 多米诺骨牌 🀫

> Forward 是顺次推倒;Backward 是从最后一张倒推谁该担多大责任。

- **forward**:从输入推到 loss(骨牌一个个倒下)
- **backward**:从 loss 倒推每个参数对错误的"贡献度"
- **链式法则**:每张骨牌只负责把"前一张推它的力"传给"后一张"
- **激活检查点**:倒骨牌前不全程录像,只在关键节点拍照,反向时按需"再倒一次"

### 类比二: 公司财报问责 🏢

> 季度亏损了(loss 大),老板要找每个部门、每个员工的责任。

- **公司组织树** = 计算图
- **总账本** = forward 的中间结果
- **CEO 把锅分给 VP,VP 分给经理,经理分给员工** = 链式法则
- **每个员工得到的"问责梯度"** = `grad`
- **激活检查点** = "今年不查所有报表了,只查季度峰值,需要时再翻流水"
- **梯度裁剪** = "再气也不能罚款超过 1 万,否则团队崩溃"

### 类比三: 烹饪还原 🍳

> Loss 是炒糊的菜,反向是"问每个步骤错在哪里"。

- **forward**:加油 → 下锅 → 翻炒 → 出锅
- **菜糊了(loss)** → 倒着问:出锅环节占 10% 责任,翻炒占 70%,油温占 20%
- **每步局部 Jacobian** = 该环节"输入味道的变化 → 输出味道的变化"敏感度
- **梯度累加**:同一道食材出现在多个步骤(残差连接) → 多处梯度相加

### 三个类比对比

| 类比 | 强调什么 | 适合记忆什么 |
| --- | --- | --- |
| 🀫 多米诺 | **方向 + 顺序** | forward / backward 的图结构 |
| 🏢 公司问责 | **梯度 = 责任** | 链式法则的分配性 |
| 🍳 烹饪还原 | **局部敏感度** | Jacobian 与 vjp |

**记忆口诀**:
> **"前向算结果,后向算责任;形状要对齐,矩阵要转置。"**

## 3.10 常见疑问

### Q1: 为什么训练显存是推理的 4-6 倍?
**A**: 推理只需权重 + 当前激活;训练还要存 **梯度 + 优化器状态(AdamW 多 2-3 倍)+ 反向需要的全部中间激活**。其中激活通常是最大头(尤其长 T 大 B)→ §3.7 的检查点正是为它而生。

### Q2: $\partial \mathcal{L}/\partial W$ 的形状一定与 $W$ 一致吗?
**A**: 是。这是反向传播的硬性约束(梯度下降需要 $W \leftarrow W - \eta g$,形状必须一致)。这一点是手算反向时的"形状校验"工具。

### Q3: detach() 和 no_grad() 有什么区别?
**A**:
- `tensor.detach()`:返回不带 grad_fn 的新张量,**仍指向同一存储**;原张量该计算图正常
- `with torch.no_grad():`:上下文管理器,**整个块内的张量都不建立计算图**,等价于关闭 autograd

推理时用 `no_grad()`,提取 embedding / target 用 `detach()`。

### Q4: 残差连接 $h + f(h)$ 的反向怎么走?
**A**: 加法的反向就是"把上游梯度复制两份分给两路输入":

$$
\frac{\partial \mathcal{L}}{\partial h}\Big|_{\text{from out}} = \frac{\partial \mathcal{L}}{\partial \text{out}} + \frac{\partial \mathcal{L}}{\partial \text{out}} \cdot \frac{\partial f}{\partial h}
$$

恒等项就是"梯度高速公路"——§09.2.2 的形式化版本。

### Q5: 为什么 backward 不会自动 zero_grad?
**A**: 设计为**累加**模式,因为很多场景需要(梯度累积、多任务 loss、ZeRO 等)。代价是必须手动 `optimizer.zero_grad()`,新手容易忘。

## 3.11 本章小结

### 核心公式

**链式法则**:

$$
\frac{dy}{dx} = \sum_i \frac{\partial y}{\partial u_i} \frac{du_i}{dx}
$$

**矩阵乘反向**:

$$
\frac{\partial \mathcal{L}}{\partial W} = X^\top \frac{\partial \mathcal{L}}{\partial Y}, \quad \frac{\partial \mathcal{L}}{\partial X} = \frac{\partial \mathcal{L}}{\partial Y} W^\top
$$

**Softmax + CE 反向**:

$$
\frac{\partial \mathcal{L}}{\partial z} = p - y
$$

**梯度裁剪**:

$$
g \leftarrow g \cdot \min(1, c / \|g\|)
$$

### 核心要点

| 要点 | 关键 |
| --- | --- |
| **链式法则** | 反向传播唯一原理 |
| **形状对齐** | 检验梯度公式正确性 |
| **计算图** | 动态图(PyTorch)易调试 |
| **训练显存** | 激活是大头,检查点能省 |
| **梯度裁剪** | LLM 训练必选 |

### 一句话总结

> **反向传播 = 在计算图上逆序应用链式法则。每个算子只需提供"局部 Jacobian × 上游梯度"(vjp),autograd 框架自动串联;训练显存中激活是大头,gradient checkpointing 是"以一次重 forward 换 √N 显存"的关键工程技巧。**
