## 摘要

LLM 是一个**条件概率模型**: $p_\theta(x_{t+1} \mid x_{\leq t})$。所有训练、采样、对齐相关的数学都在概率与信息论的语言里:

- **MLE**: 训练等价于最大化训练数据在模型分布下的对数似然
- **交叉熵**: 训练 loss 的真名
- **KL 散度**: 衡量两个分布差距,SFT / DPO / RLHF / 蒸馏的统一工具
- **Softmax**: 把任意实数向量翻译成"合法的离散分布"
- **采样三件套**: 温度 / Top-k / Top-p 控制生成多样性

本章按"读 / 写 LLM 论文够用"的级别梳理,不做严格测度论。

## 2.1 概率三件套:分布、期望、方差

### 2.1.1 离散概率分布

随机变量 $X$ 取离散值 $\{a_1, a_2, \ldots\}$,分布:

$$
p(a_i) \geq 0, \quad \sum_i p(a_i) = 1 \tag{2.1}
$$

LLM 中每一步预测的"下一个 token"就是一个**取值于词表 $V$ 的离散分布**,长度 $V$ 的概率向量。

### 2.1.2 连续概率密度

连续 $X$ 用密度 $f(x)$ 代替:

$$
f(x) \geq 0, \quad \int f(x)\,dx = 1 \tag{2.2}
$$

LLM 中权重的初始化、激活的分布近似都用连续分布(主要是高斯)描述。

### 2.1.3 期望与方差

$$
\mathbb{E}[X] = \sum_i a_i \, p(a_i), \quad \text{Var}(X) = \mathbb{E}[(X - \mathbb{E}[X])^2] \tag{2.3}
$$

LLM 中:

- 梯度下降 = 在 mini-batch 上**估计期望梯度**
- 训练稳定性 = 控制**激活方差**(归一化的作用)

## 2.2 常用分布:伯努利 / 多项式 / 高斯 / 类别

### 2.2.1 伯努利 / 类别(Categorical)

**伯努利**($K=2$):$p(X=1) = \pi$, $p(X=0) = 1-\pi$。

**类别**($K\geq 2$):$p(X=k) = \pi_k$, $\sum_k \pi_k = 1$。

LLM 每步输出就是 $V$ 类的 Categorical。

### 2.2.2 多项式(Multinomial)

$N$ 次独立 Categorical 试验,每类计数 $n_k$:

$$
p(n_1, \ldots, n_K) = \frac{N!}{\prod n_k!} \prod \pi_k^{n_k} \tag{2.4}
$$

LLM 中较少直接出现,但理解 cross-entropy 的来源需要它。

### 2.2.3 高斯(Normal)

$$
\mathcal{N}(x; \mu, \sigma^2) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\!\left(-\frac{(x-\mu)^2}{2\sigma^2}\right) \tag{2.5}
$$

- 权重初始化:$W_{ij} \sim \mathcal{N}(0, \sigma^2)$
- 中心极限定理:大量独立同分布求和 → 高斯 → 矩阵乘法激活分布近似高斯 → 用 RMSNorm 控住方差

### 2.2.4 一句话总结表

| 分布 | LLM 场景 |
| --- | --- |
| 伯努利 | 二分类标签(如毒性检测) |
| Categorical(类别) | **每步 token 预测的 ground truth** |
| 多项式 | Categorical 的批扩展 |
| 高斯 | **权重初始化 / 激活分布近似** |

## 2.3 条件概率与贝叶斯

### 2.3.1 条件概率

$$
p(A \mid B) = \frac{p(A, B)}{p(B)} \tag{2.6}
$$

LLM 本质是 $p_\theta(x_{t+1} \mid x_{\leq t})$——给定历史 token,下一 token 的条件分布。

### 2.3.2 贝叶斯公式

$$
p(\theta \mid \mathcal{D}) = \frac{p(\mathcal{D} \mid \theta) p(\theta)}{p(\mathcal{D})} \tag{2.7}
$$

- 后验 $p(\theta \mid \mathcal{D})$:给定数据后,参数的分布
- 似然 $p(\mathcal{D} \mid \theta)$:参数固定时,数据出现的概率
- 先验 $p(\theta)$:对参数的"主观信念"

LLM 训练取**MLE**(只最大化似然,等价于先验均匀)。Bayes 视角在 LoRA、贝叶斯神经网络中偶有用,但主线只用 MLE。

### 2.3.3 链式法则与序列建模

$$
p(x_1, x_2, \ldots, x_T) = \prod_{t=1}^{T} p(x_t \mid x_{<t}) \tag{2.8}
$$

这是**自回归语言模型**的根本依据:整段序列概率 = 每步条件概率的乘积。

## 2.4 最大似然估计(MLE)

### 2.4.1 定义

给数据集 $\mathcal{D} = \{x^{(i)}\}_{i=1}^N$,选 $\theta$ 使数据出现概率最大:

$$
\hat\theta_{\text{MLE}} = \arg\max_\theta \prod_{i=1}^{N} p_\theta(x^{(i)}) = \arg\max_\theta \sum_{i=1}^{N} \log p_\theta(x^{(i)}) \tag{2.9}
$$

求和比连乘更易优化、数值更稳定。

### 2.4.2 LLM 训练 = 序列 MLE

代入链式法则 (2.8):

$$
\mathcal{L}(\theta) = -\frac{1}{N} \sum_{i=1}^{N} \sum_{t=1}^{T_i} \log p_\theta(x^{(i)}_t \mid x^{(i)}_{<t}) \tag{2.10}
$$

最小化 $\mathcal{L}$ 等价于最大化 (2.9) → **训练 LLM = 在所有训练 token 上做条件 MLE**。

### 2.4.3 与交叉熵的等价

后面 §2.5.4 会看到 $\mathcal{L}$ 就是逐 token 的 **cross-entropy** 平均——所以"训练 LLM"、"最大化对数似然"、"最小化 cross-entropy" 是同一件事的三种说法。

## 2.5 信息论:熵、交叉熵、KL 散度

### 2.5.1 熵(Shannon Entropy)

$$
H(p) = -\sum_x p(x) \log p(x) \tag{2.11}
$$

直觉:**编码这个分布的平均码长**。$\log$ 用 2 为底单位是 **bit**,用 $e$ 为底单位是 **nat**。

- 均匀分布 $H = \log K$(最大)
- 退化为单点 $H = 0$(最小)

### 2.5.2 交叉熵(Cross-Entropy)

$$
H(p, q) = -\sum_x p(x) \log q(x) \tag{2.12}
$$

**含义**: 用分布 $q$ 编码来自 $p$ 的样本需要的平均码长。

### 2.5.3 KL 散度

$$
\text{KL}(p \| q) = \sum_x p(x) \log \frac{p(x)}{q(x)} = H(p, q) - H(p) \tag{2.13}
$$

性质:

- $\text{KL}(p \| q) \geq 0$,等号当且仅当 $p = q$
- **非对称**: $\text{KL}(p \| q) \neq \text{KL}(q \| p)$
- 衡量"$q$ 离 $p$ 多远"(以 $p$ 视角)

### 2.5.4 把 MLE 化成 Cross-Entropy

经验数据分布 $\hat p(x) = \tfrac{1}{N}\sum_i \mathbb{1}[x = x^{(i)}]$,代入 (2.12):

$$
H(\hat p, p_\theta) = -\frac{1}{N}\sum_i \log p_\theta(x^{(i)}) \tag{2.14}
$$

这正是 (2.10) 的形式 → **训练 loss = cross-entropy = -log-likelihood**,三者等价。

进一步:

$$
\arg\min_\theta H(\hat p, p_\theta) = \arg\min_\theta \text{KL}(\hat p \| p_\theta)
$$

因为 $H(\hat p)$ 与 $\theta$ 无关。所以 MLE 也等价于"最小化 KL 到经验分布"。

### 2.5.5 LLM 评估指标:PPL / BPT / BPB

- **Perplexity**: $\text{PPL} = \exp(H) = \exp(\text{CE})$,"平均犹豫的 token 数"
- **Bits per Token (BPT)**: $\text{CE} / \ln 2$
- **Bits per Byte (BPB)**: 与 tokenizer 无关的归一化版本,$\text{BPB} = \text{BPT} \cdot (\text{token/byte 平均比})$

跨 tokenizer 比较模型时**只能比 BPB**,否则不公平。

## 2.6 Softmax 与 Logits 的概率含义

### 2.6.1 Softmax 定义

把任意实数向量 $z \in \mathbb{R}^V$ 映射成合法概率分布:

$$
p_i = \frac{\exp(z_i)}{\sum_{j=1}^{V} \exp(z_j)} \tag{2.15}
$$

性质:

- $p_i > 0$ 且 $\sum p_i = 1$ → 合法分布
- 平移不变:$\text{softmax}(z + c) = \text{softmax}(z)$,可减去 $\max z$ 防 overflow
- 平滑可微:配合 cross-entropy,梯度形式简洁

### 2.6.2 Softmax 的"温度"

$$
p_i = \frac{\exp(z_i / \tau)}{\sum_j \exp(z_j / \tau)} \tag{2.16}
$$

- $\tau \to 0$:接近 one-hot(argmax)
- $\tau = 1$:原始 softmax
- $\tau \to \infty$:接近均匀分布

LLM 采样时 $\tau$ 控制"创造力 vs 准确性"。

### 2.6.3 Logits 的几何

Logits 是 LM head 输出,直观上就是"每个 token 的得分"。Softmax 把这种 unbounded 得分转成 bounded 概率。

注意: **Softmax + Cross-Entropy 的梯度刚好是 $\hat p - p$(预测减真实)**,这就是为什么实现上常融合成 `F.cross_entropy` 单个 kernel(数值更稳)。

## 2.7 采样:温度、Top-k、Top-p

### 2.7.1 Greedy(温度 0)

每步选概率最大的 token。**确定性,但容易陷入重复**。

### 2.7.2 温度采样

按 (2.16) 软化分布后多项式采样。常用 $\tau \in [0.7, 1.2]$。

### 2.7.3 Top-k

只保留概率最大的 $k$ 个 token,重新归一化后采样。截掉长尾低质 token。

### 2.7.4 Top-p (Nucleus)

按概率降序排序,累积到 $p$(如 0.9)为止,只在这部分采样。$k$ 自适应。

### 2.7.5 实战配方

| 场景 | 推荐 |
| --- | --- |
| 代码 / 推理(要确定) | $\tau = 0$(greedy) 或 0.2 |
| 对话(平衡) | $\tau = 0.7$,top_p = 0.9 |
| 创意写作 | $\tau = 1.0$+,top_p = 0.95 |
| 蒸馏 / RL 探索 | 较大 $\tau$ + Top-k 限制 |

详细工程在 §20。

## 2.8 完整代码实现

```python
import torch
import torch.nn.functional as F

# 1. softmax & log_softmax
z = torch.tensor([2.0, 1.0, 0.5])
p = F.softmax(z, dim=-1)
log_p = F.log_softmax(z, dim=-1)
print(p, log_p.exp())   # 应该一致

# 2. cross-entropy(手写 vs 官方)
target = torch.tensor([0])           # 类别 0
ce_manual = -log_p[target]
ce_torch  = F.cross_entropy(z.unsqueeze(0), target)
print(ce_manual.item(), ce_torch.item())   # 完全一致

# 3. KL 散度(p 真分布,q 模型)
p_true = torch.tensor([0.7, 0.2, 0.1])
q_model = F.softmax(z, dim=-1)
kl = (p_true * (p_true / q_model).log()).sum()
kl_torch = F.kl_div(q_model.log(), p_true, reduction='sum')
print(kl.item(), kl_torch.item())

# 4. 温度采样
def sample(logits, temperature=1.0, top_k=None, top_p=None):
    logits = logits / temperature
    if top_k is not None:
        v, _ = torch.topk(logits, top_k)
        logits[logits < v[-1]] = -float('inf')
    if top_p is not None:
        sorted_logits, idx = torch.sort(logits, descending=True)
        probs = F.softmax(sorted_logits, dim=-1)
        cum = probs.cumsum(dim=-1)
        mask = cum > top_p
        mask[..., 1:] = mask[..., :-1].clone()
        mask[..., 0] = False
        sorted_logits[mask] = -float('inf')
        logits.scatter_(-1, idx, sorted_logits)
    probs = F.softmax(logits, dim=-1)
    return torch.multinomial(probs, 1)
```

## 2.9 LLM 中的应用速查

| 场景 | 用到的工具 |
| --- | --- |
| 训练 loss | Cross-Entropy(=负对数似然) |
| 评估困惑度 | $\text{PPL} = \exp(\text{CE})$ |
| 蒸馏 | $\text{KL}(p_{\text{teacher}} \| p_{\text{student}})$ |
| RLHF 的 reward shaping | $-\beta \cdot \text{KL}(\pi_\theta \| \pi_{\text{ref}})$ |
| DPO loss | 直接基于偏好对的 logistic loss,数学上仍以 KL 为约束 |
| 采样 | softmax + 温度 + Top-k / Top-p |
| Mixture-of-Experts 负载均衡 | KL 到均匀分布作为辅助 loss |
| Label smoothing | 把 one-hot 与均匀分布按 $(1-\epsilon)$ 混合,减少过拟合 |

## 🎨 2.9.A 通俗类比合集

### 类比一: 押注游戏 🎲

> 训练 = 反复让模型对"下一个字"押注,押对得奖,押错挨罚。

- **概率分布** = 你心里对各种结果的"押注比例"
- **MLE** = 用全部历史押注记录,反推"押注策略"
- **Cross-Entropy** = 把"押错"的痛苦量化(对正确答案给低概率 = 大量罚款)
- **KL 散度** = 比较两个赌徒的策略偏差有多大
- **温度** = 你押注时"是更冒险(高 τ)还是更保守(低 τ)"

### 类比二: 编码电报 📨

> 信息论原意:用最短电报传递文字。

- **熵** = 这个文字源的"理论最短平均电报长度"
- **交叉熵** = 你用了某套不优的编码方案,实际平均长度
- **KL** = 你的方案比最优长多少
- **PPL** = "解码这一段平均要犹豫几次" = $2^{\text{bits}}$

### 类比三: 多人猜谜 🧩

> 几个朋友一起猜你脑中的词,每人有一套"猜词策略"(概率分布)。

- **真实分布 $p$** = 你心里那个词的"真实概率"
- **每个朋友的策略 $q_i$** = 各自的猜测分布
- **Cross-Entropy** = 哪个朋友平均要猜几次才能猜中(越小越准)
- **KL** = 哪个朋友的策略离你心里最近
- **Softmax + 温度** = 朋友 "毕竟还是要随机一点" 的自由度调节

### 三个类比对比

| 类比 | 强调什么 | 适合记忆什么 |
| --- | --- | --- |
| 🎲 押注 | **奖罚 = loss** | MLE / Cross-Entropy 等价 |
| 📨 电报 | **平均码长** | 熵 / KL / PPL 三件套 |
| 🧩 猜谜 | **多分布比较** | 软化 vs 锐化 |

**记忆口诀**:
> **"MLE = 最大对数似然 = 最小交叉熵 = 最小 KL 到经验分布,四词一义。"**

## 2.10 常见疑问

### Q1: KL 不对称,我应该用 $\text{KL}(p\|q)$ 还是 $\text{KL}(q\|p)$?
**A**: 看你想要哪种"覆盖":
- $\text{KL}(p\|q)$:**mode-covering**,$q$ 必须覆盖所有 $p>0$ 的地方,否则代价无穷 → 用于训练时(MLE)
- $\text{KL}(q\|p)$:**mode-seeking**,$q$ 可以"挑个峰扎进去" → 用于变分推断、某些 RL
LLM 训练几乎都用 $\text{KL}(\hat p \| p_\theta)$,即 MLE。

### Q2: 为什么 cross-entropy 不直接用 `(-log(p)).mean()` 而要 `F.cross_entropy`?
**A**: 数值稳定。`F.cross_entropy` 内部做 `log_softmax + nll`,避免单独 `softmax` 后取 log 导致的 underflow(指数差极大时)。

### Q3: 温度 $\tau$ 和正则化里的"温度"是同一回事吗?
**A**: 数学形式一样($\exp(z/\tau)$ 归一)但用途不同。采样里改的是输出分布,蒸馏 / 对比学习里用温度调"相似度集中度"。理解原理一致,使用场景不同。

### Q4: 为什么 LLM 一直用 MLE,不用 Bayes?
**A**: 工程上 Bayes 后验对大模型成本极高(MCMC、变分都不易 scale)。但 LoRA + KL 正则、ensembling 等技巧可看作近似 Bayes。研究方向有,但生产用 MLE。

### Q5: PPL 和 BPB 我看哪个?
**A**:
- 同一模型不同检查点比较:PPL 简单直接
- 不同 tokenizer / 不同模型比较:**只能用 BPB**(否则中文 token 多的模型看似 PPL 高其实信息量相同)

## 2.11 本章小结

### 核心公式

**MLE**:

$$
\hat\theta = \arg\max_\theta \sum_i \log p_\theta(x^{(i)})
$$

**Cross-Entropy**:

$$
H(p, q) = -\sum_x p(x) \log q(x)
$$

**KL**:

$$
\text{KL}(p \| q) = H(p, q) - H(p)
$$

**Softmax(带温度)**:

$$
p_i = \frac{e^{z_i / \tau}}{\sum_j e^{z_j / \tau}}
$$

### 核心要点

| 要点 | 关键 |
| --- | --- |
| **训练** | MLE = Cross-Entropy = -log-likelihood |
| **评估** | PPL = exp(CE),跨 tokenizer 用 BPB |
| **蒸馏 / 对齐** | KL 散度是统一工具 |
| **采样** | 温度 + Top-k / Top-p |
| **数值** | logits 减最大值再 softmax,fp32 累加 |

### 一句话总结

> **LLM 是条件概率模型,训练是最大化训练数据的对数似然(= 最小化交叉熵 = 最小化 KL 到经验分布);Softmax 把 logits 翻译为合法分布,温度控制采样的"确定 vs 创造"。这一套概率信息论语言贯穿 SFT、RLHF、蒸馏、采样的全部环节。**

## 2.13 实战练习

### 练习 1 (★)

在 PyTorch 中验证 `F.cross_entropy(logits, target)` 与 `F.nll_loss(F.log_softmax(logits, -1), target)` 完全一致。

### 练习 2 (★)

固定 logits 不变,绘制 $\tau \in \{0.1, 0.5, 1.0, 2.0, 10.0\}$ 时 softmax 分布的形状变化,直观感受温度。

### 练习 3 (★★)

从 HuggingFace 加载 GPT-2,对 WikiText-2 测试集计算 cross-entropy 与 perplexity,验证与公开数字一致。

### 练习 4 (★★)

实现 Top-k 与 Top-p 采样,用相同模型生成 100 句话,统计:重复率、唯一 1-gram 数,看温度 / 截断如何影响多样性。

### 练习 5 (★★★)

蒸馏小练习:用一个大模型当 teacher,小模型当 student,loss = $\alpha \cdot \text{CE}(p_\text{data}, p_s) + (1-\alpha) \cdot \tau^2 \cdot \text{KL}(p_t^\tau \| p_s^\tau)$,验证小模型收敛速度变化。
