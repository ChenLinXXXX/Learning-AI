## 摘要

多模态 LLM 的核心是把"图像 / 视频 / 音频"也变成 token,**让它们和文本 token 一起进入 LLM**。具体做法:

- 用一个 **encoder**(ViT / SigLIP / AST 等)把模态信号变成连续向量序列
- 用一个轻量 **projector**(MLP / Q-Former / Resampler)把这些向量"翻译"到 LLM 的隐空间
- LLM 本身几乎不改,只在多模态数据上 SFT(+ 偏好对齐)

读完你能解释 CLIP 怎么训、LLaVA 为什么只用一个 MLP 就能 work、Qwen-VL 的 any-resolution 怎么处理任意分辨率,以及视频 / 音频是怎么挤进同一 LLM 的。

## 29.1 总体框架:Encoder + Projector + LLM

```text
图像 / 音频 / 视频
     │
     ▼
[模态 Encoder]  ←(预训练好,通常冻结或部分微调)
     │   visual_tokens (N_v, d_v)
     ▼
[Projector]    ←(把 d_v → d_model,并可减少 token 数)
     │   text-like tokens (N_v', d_model)
     ▼
[LLM(§10)] ←── 与文本 token 拼接 →  [text tokens]
     │
     ▼
   logits / 生成
```

三件套关键:

| 组件 | 角色 | 是否训练 |
| --- | --- | --- |
| **Encoder** | 模态 → 高维向量序列 | 通常冻结(预训练已经好) |
| **Projector** | 把 encoder 输出对齐到 LLM 隐空间 | 必训(对齐两侧) |
| **LLM** | 主推理 | SFT 阶段微调或冻结 |

## 29.2 视觉编码器:ViT / SigLIP / DINOv2

### 29.2.1 Vision Transformer(ViT,Dosovitskiy 2020)

把图像切成 patch(典型 14×14 / 16×16 像素),每 patch 当一个 token,送进标准 Transformer Encoder:

```text
图像 H×W×3
  → 切 N = HW/p² 个 patch
  → 每 patch 展平 + linear → token
  → + 位置编码 + [CLS] token
  → Transformer × L
  → 输出 N+1 个 token 向量
```

1024×1024 + patch=14 → 5329 个 patch token。**这数字直接关系到后续 LLM 序列长度**。

### 29.2.2 CLIP-ViT、SigLIP

CLIP-ViT 把 ViT 用对比学习预训练(下节);**SigLIP**(2023)把 CLIP 的 softmax loss 换成 sigmoid loss,实证稳定性 + 性能更好,**LLaVA-1.6+ / Qwen-VL / DeepSeek-VL 都用 SigLIP**。

### 29.2.3 DINOv2 等自监督

DINOv2(Meta)用纯自监督训出的 ViT,**不依赖文本配对**,在多任务上比 CLIP-ViT 强。部分多模态 LLM 用 DINOv2 + SigLIP 双 encoder。

## 29.3 对比预训练:CLIP

### 29.3.1 思想

Radford et al., 2021。**给一批 (image, caption) 对,让正确配对的 cosine 相似度最高**。

- Image encoder $f_v$:ViT
- Text encoder $f_t$:Transformer
- 共同嵌入到 $d$ 维空间,L2 归一化

### 29.3.2 InfoNCE Loss

batch 内 $N$ 对 $(I_i, T_i)$:

$$
\mathcal{L} = -\frac{1}{2N} \sum_{i=1}^{N}\!\Bigg[\log \frac{e^{\langle v_i, t_i\rangle / \tau}}{\sum_{j} e^{\langle v_i, t_j\rangle / \tau}} + \log \frac{e^{\langle v_i, t_i\rangle / \tau}}{\sum_{j} e^{\langle v_j, t_i\rangle / \tau}} \Bigg] \tag{29.1}
$$

每个 i 既被当 image-side query 也被当 text-side query。$\tau$ 是可学习温度。

### 29.3.3 SigLIP 的改动

把 softmax 换成 sigmoid:每对 (i, j) 独立判"匹配 vs 不匹配",**不用做 batch 内归一化**:

$$
\mathcal{L} = \sum_{i,j} \log \big(1 + \exp((b - \langle v_i, t_j \rangle / \tau) \cdot z_{ij})\big) \tag{29.2}
$$

$z_{ij} = +1$ if $i=j$, else $-1$。优势:对 batch size 不敏感,小 batch 也能好好训。

### 29.3.4 CLIP 的能力

- Zero-shot classification:把类别名做 caption,取相似度最高的类
- 图像检索 / 文本检索
- 作为下游多模态 LLM 的视觉 encoder

CLIP 与 SigLIP 是 2021-2025 视觉 encoder 的事实标准。

## 29.4 早期生成式:Flamingo / BLIP-2

### 29.4.1 Flamingo(DeepMind 2022)

- 视觉:NF-Net / ViT
- 在 LLM 中**插入 cross-attention 层**,让 text token 能看图
- 训练时只训新增 cross-attn,LLM 主体冻结
- 适合 few-shot in-context vision learning

工程复杂(改了 LLM 结构),后续被更轻的方案替代。

### 29.4.2 BLIP-2(Salesforce 2023)

引入 **Q-Former**:一个小 Transformer,以**少量可学习 query**为输入,经 cross-attn 抽取图像特征 → 输出 N 个 LLM-compatible token:

```text
ViT 输出(几百 token)
  ┌─────────────────┐
  │  Q-Former       │  ← 32 个 learnable queries
  │  (cross-attn)   │
  └─────────────────┘
       │
       ▼ 32 个高质量 token
   送进 LLM
```

优点:**显著压缩视觉 token 数**;缺点:Q-Former 训练较复杂,后续 LLaVA 系列用更简单方案。

## 29.5 主流方案:LLaVA / Qwen-VL / InternVL

### 29.5.1 LLaVA(Liu et al., 2023)

> 极简方案:**Encoder 输出 → 一层 / 两层 MLP → LLM token**。

```text
SigLIP / CLIP-ViT
   ↓ 576 / 729 token(d_v=1024)
[MLP(d_v → d_model)]
   ↓ 576 / 729 token(d_model)
LLM(LLaMA / Vicuna / Qwen)
```

训练分两阶段:

1. **Pre-train(对齐)**:冻结 encoder 与 LLM,只训 MLP。在 ~600k 图文对上训。
2. **Visual Instruction Tuning**:解冻 LLM(或部分),用 ~150k 多模态对话数据 SFT。

LLaVA-1.5(2023)就在 8×A100 上几小时训完,**架构极简、效果惊艳**,成为开源多模态默认范式。

### 29.5.2 为什么 MLP 就够?

直觉:CLIP/SigLIP 训出的视觉空间已经"语义化",**不需要复杂网络与 LLM 隐空间对齐,简单线性 + 非线性即可**。少参 + 少训练 = 更稳。

### 29.5.3 Qwen-VL / InternVL / DeepSeek-VL

主流方案都是 LLaVA 范式的扩展:

- **Qwen-VL**:用 ViT + 1024 queries 的 Resampler(类 Q-Former 但更简单);支持中文与文档
- **InternVL**:大 ViT(6B+) + 大 LLM,极致 scale up
- **DeepSeek-VL**:SigLIP-Large + 2-MLP projector + DeepSeek-LLM base,支持 1024×1024
- **LLaMA-3.2 Vision**:Meta 官方多模态

它们都遵循"Encoder + Projector + LLM"的三段式。

## 29.6 高分辨率:Any-Resolution / 分块

### 29.6.1 痛点

LLaVA-1.5 默认 336×336 输入,576 token 进 LLM。

- OCR / 文档 / UI 截图需要更高分辨率 → 直接放大会让 ViT 的位置编码越界
- 强行 resize 到 336 损失细节,文字看不清

### 29.6.2 Any-Resolution(LLaVA-1.6 / Qwen-VL / InternVL2)

把大图**切成多个小块**,每块独立过 ViT,再把所有 token 拼起来交给 LLM:

```text
1024×1024 图像
  → 切 9 个 336×336 块 + 1 张缩略图(全局)
  → 10 × 576 = 5760 个 token
  → 全部送进 LLM
```

主流实现还会:

- 选合适 grid(如 2×3、3×3)使长宽比接近
- 加 newline token / 分块 token 帮 LLM 理解空间结构
- 全局缩略图保留整体语义

### 29.6.3 代价与折中

- token 数大涨(5k+),对 LLM 上下文与 KV 压力大
- 训练时多分辨率 sampling 增加复杂度

很多模型给出"低/中/高分辨率三档"让用户选,平衡成本与精度。

### 29.6.4 Native-Resolution(Qwen2.5-VL 等)

把"any-resolution"推到极致:不强制 patch grid 对齐,模型直接吃任意像素,文档 / OCR 性能显著提升。

## 29.7 视频与音频

### 29.7.1 视频

最简单:每秒抽 1-8 帧,每帧当独立图像 ViT 处理,token 拼成长序列。

进阶:

- 加 **temporal attention**(跨帧关注)
- 时空位置编码
- 视频专用 encoder(VideoMAE、UMT)

代表模型:LLaVA-Video / Qwen2-VL Video / Gemini Video。

### 29.7.2 音频

- 把音频转 mel-spectrogram(2D 图像形式)再走 ViT 风格 encoder
- 或用 Whisper-like encoder(Audio Spectrogram Transformer / AST)
- 输出 token 送进 LLM

代表:Qwen2-Audio、Whisper-LLM、GPT-4o Voice。

### 29.7.3 多模态融合

近年趋势:**所有模态 → 统一 token 序列 → 一个 LLM**(早期分支式架构正在退潮)。如 GPT-4o、Gemini 在结构上接近"全模态统一序列"。

## 29.8 训练流程:两阶段 / 三阶段

### 29.8.1 LLaVA-1.5 两阶段

1. **Stage 1 — Vision-Language Alignment**:冻结 encoder + LLM,只训 projector。数据:CC3M、LAION 等图文对(~600k)
2. **Stage 2 — Visual Instruction Tuning**:解冻 LLM(全参或 LoRA),数据:多模态 instruction(~150k,GPT-4 合成的图像描述对话 / VQA / OCR 等)

### 29.8.2 Qwen-VL / InternVL 三阶段

1. **Vision-Language Pretraining(大规模)**:数十 B 图文对,冻结 LLM
2. **Multi-Task Pretraining**:加 OCR、grounding、视频等任务,继续大规模
3. **Supervised Fine-Tuning**:高质量 instruction,通用 chat + vision

### 29.8.3 多模态 RLHF / DPO

近年也开始在多模态上做 RLHF(避免幻觉、改进 OCR 准确性)。难点:reward 设计在视觉场景更难;数据成本高。

## 29.9 完整代码:LLaVA 风格 forward

```python
import torch
import torch.nn as nn

class VisionProjector(nn.Module):
    def __init__(self, d_v: int, d_model: int):
        super().__init__()
        self.proj = nn.Sequential(
            nn.Linear(d_v, d_model),
            nn.GELU(),
            nn.Linear(d_model, d_model),
        )
    def forward(self, x):  # x: (B, N_v, d_v)
        return self.proj(x)


class MMLM(nn.Module):
    """
    LLaVA 风格:vision encoder + projector + LLM
    """
    def __init__(self, vision_encoder, llm, d_v, d_model):
        super().__init__()
        self.vision = vision_encoder      # 冻结
        for p in self.vision.parameters(): p.requires_grad_(False)
        self.projector = VisionProjector(d_v, d_model)
        self.llm = llm

    def forward(self, pixel_values, input_ids, image_token_id):
        """
        pixel_values: (B, 3, H, W)
        input_ids:    (B, T),其中 image_token_id 表示"图像占位 token"
        """
        # 1. 视觉 token
        with torch.no_grad():
            vis = self.vision(pixel_values)         # (B, N_v, d_v)
        vis = self.projector(vis)                   # (B, N_v, d_model)

        # 2. 文本 embedding
        text_emb = self.llm.tok_emb(input_ids)      # (B, T, d_model)

        # 3. 把 <image> 占位符位置替换成视觉 token
        out = []
        for b in range(input_ids.size(0)):
            text_b = text_emb[b]
            vis_b  = vis[b]
            pos = (input_ids[b] == image_token_id).nonzero().squeeze(-1)
            assert len(pos) == 1, "假设 prompt 中只有 1 张图"
            i = pos.item()
            merged = torch.cat([text_b[:i], vis_b, text_b[i+1:]], dim=0)
            out.append(merged)
        out = torch.nn.utils.rnn.pad_sequence(out, batch_first=True)

        # 4. 继续走 LLM(已带位置编码 / KV cache)
        # 注:实际实现要重算 position_ids,因为序列长度变化
        logits = self.llm.forward_embeds(out)
        return logits
```

> 简化版,缺少:多图、任意分辨率切块、KV cache 处理。生产参考 LLaVA / Qwen-VL / DeepSeek-VL 源码。

## 29.10 评测与挑战

### 29.10.1 主流 benchmark

- **VQAv2 / GQA / OK-VQA**:视觉问答

### 29.10.2 现存挑战

| 问题 | 描述 |
| --- | --- |
| **视觉幻觉** | "看到不存在的物体" |

## 🎨 29.10.A 通俗类比合集

### 类比一: 哑剧翻译员 🎭

> Encoder 看哑剧,projector 翻译,LLM 听翻译后讲故事。

- **Encoder = 哑剧观众**(默默看完每幕,记下"动作向量")
- **Projector = 翻译员**(把动作描述成 LLM 听得懂的语言)
- **LLM = 讲故事的作家**(基于翻译后的描述写文章 / 回答问题)
- **CLIP 预训练 = 翻译员长期接受配对训练**(图配字幕),练就高质量同传能力
- **LLaVA 范式** = 翻译员是个小 MLP,**简单但够用**

### 类比二: 公司里的"双语秘书" 🏢

> CEO(LLM)只懂中文,但客户(图像)说英文。

- **ViT encoder = 客户讲英文** → 输出英文要点
- **Projector = 双语秘书** → 翻译成中文 PPT
- **LLM = CEO** → 看 PPT 做决策回复
- **CLIP** = 公司请秘书时**先用海量双语对照培训**
- **High-Res tiling** = 客户拿来 100 页文档,秘书分册翻译再汇总

### 类比三: 给盲人讲电影 🎬

> 字面对应"视觉到语言"。

- **Encoder = 现场导览员**,看银幕做笔记
- **Projector = 即时翻译者**,把笔记翻成故事性中文
- **LLM = 听故事的盲人朋友**,根据故事回答你的问题
- **Any-resolution** = 银幕太大时,导览员**分块描述**,最后合一份完整描述
- **多模态训练** = 长期合作演练,导览员与朋友越来越默契

### 三个类比对比

| 类比 | 强调什么 | 适合记忆什么 |
| --- | --- | --- |
| 🎭 哑剧 | **跨语言对齐** | CLIP 对比预训练 |
| 🏢 双语秘书 | **角色协作** | 三段式架构 |
| 🎬 给盲人讲 | **细节传递** | High-Res 分块 |

**记忆口诀**:
> **"图像走 ViT,token 进 MLP,合并文本喂 LLM;CLIP 对比预训练,LLaVA 简单两阶段。"**

## 29.11 常见疑问

### Q1: 为什么图像 token 数远多于文本 token?
**A**: 一张 336×336 图像 ~ 576 token,大约等于 500 字英文。这是当前架构的代价之一。Q-Former / Resampler 旨在压缩,Any-Res 则反向 →"用更多 token 换细节"。

### Q2: 我自己训多模态难在哪?
**A**:
- 数据(高质量图文对)
- 训练稳定性(LLM 微调 + projector 同步)
- 评测(benchmark 容易过拟合)
- 多模态对齐税(影响纯文本能力)
社区已开源 LLaVA / TRL / Open-VLM 等框架,从这些起步最稳。

### Q3: 一张图占多少 KV?
**A**: 与 token 数一样多。LLaVA-1.5 一张图 ~576 KV;Any-Res 高分辨率可达数千。所以 KV cache 优化(§19)对多模态尤为重要。

### Q4: 视频效果普遍不好?
**A**: 是的。时间长 → token 多 → context 爆 + 训练数据稀缺。GPT-4o / Gemini 已显著领先,开源还在追。

### Q5: 多模态模型能直接生图吗?
**A**: 一般不能(LLaVA / Qwen-VL 是"vision-in language-out")。"vision in + image out" 需要额外解码器(扩散或自回归 image token),如 GPT-4o image generation / Gemini Native Image / Janus 系列。

## 29.12 本章小结

### 核心架构

```text
Image / Video / Audio
   ↓ Encoder(ViT / SigLIP / Whisper)
   ↓ Projector(MLP / Q-Former / Resampler)
   ↓ LLM(与文本 token 拼接)
   ↓ Logits → 文本输出
```

### 核心要点

| 要点 | 关键 |
| --- | --- |
| **三段式** | Encoder + Projector + LLM |
| **CLIP / SigLIP** | 视觉 encoder 主流,对比预训练 |
| **LLaVA 极简** | 一个 MLP + 两阶段 SFT |
| **Any-Resolution** | 切块支持高分辨率 / 文档 / OCR |
| **挑战** | 幻觉 / 小字 / 视频长程 / 多模态税 |

### 一句话总结

> **多模态 LLM 的核心是把图像 / 视频 / 音频先用专用 encoder 编成向量序列,再用一个轻量 projector 翻译进 LLM 的隐空间,与文本 token 一起送进同一个 LLM;CLIP/SigLIP 是事实标准的视觉编码器,LLaVA 范式(Encoder + MLP + LLM + 两阶段 SFT)是开源主流,Any-Res 切块支持高分辨率与文档场景。**

## 29.14 实战练习

### 练习 1 (★)

用 HuggingFace `transformers` 加载 LLaVA-1.5-7B,做一张图的描述任务,记录 token 占比。

### 练习 2 (★★)

把 SigLIP 的 sigmoid loss 实现一遍,验证 §29.3.3 公式;对比 InfoNCE 的训练稳定性(小 batch 下)。

### 练习 3 (★★)

用 LLaVA 训练数据(LLaVA-1.5 instruct),冻结 encoder + LLM,只训一个 2 层 MLP projector,验证收敛。

### 练习 4 (★★★)

实现简化版 Any-Resolution:1024 图像切 2×2 + 1 缩略图,共 5 张 336 图,每张走 ViT,token 拼接送进 LLM。在 DocVQA 上对比单图版的差异。

### 练习 5 (★★★)

设计幻觉评测:对 100 张测试图各问"图中是否有 cat?",其中 50 张有 50 张没有,记录 false positive 率。讨论怎么用 RLHF / DPO 改进。
